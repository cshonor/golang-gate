什么是泛型？
 
泛型允许你编写可复用、类型安全、高性能的代码。
它通过使用类型参数  [T] ，代替具体的类型（如  int 、 string 、 struct ）。
 
Go 1.18 之前
 
我们只能用：
 
-  interface{} ：丢失类型安全，容易出运行时错误
- 反射（reflect）：性能差、不安全、代码难读
- 为每种类型重复写几乎一样的代码
 
Go 1.18 之后
 
泛型解决了这些问题：
 
- 类型安全：编译期就检查类型，不会运行时崩溃
- 高性能：性能和直接写具体类型代码几乎一样
- 代码复用：一个函数/结构体支持所有类型
- 代码更简洁：无冗余、无反射
 
核心概念
 
1. 类型参数： [T any] 
-  T  是任意类型的占位符
2. 类型约束：限制  T  可以是什么类型
- 例如： Number 、 comparable 
3. 泛型函数 / 泛型结构体 / 泛型类型
 
为什么泛型对 CDN 开发很重要？
 
- 编写高性能缓存（通用 LRU、 Cache[K,V] ）
- 构建可复用网络工具，不使用反射
- 在高并发场景下提升代码稳定性
- 适配现代 Go 标准库（ slices 、 maps ）
 
简单示例
 
go  
// 泛型函数：支持任意类型
func Print[T any](v T) {
    fmt.Println(v)
}

func main() {
    Print(10)
    Print("hello")
    Print(struct{ name string }{"CDN"})
}
 
 
核心总结
 
泛型 = 可复用 + 类型安全 + 高性能
它是现代 Go 编写高质量库和高性能系统（如 CDN 边缘节点）的标准方式。

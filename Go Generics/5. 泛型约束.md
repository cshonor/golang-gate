5. Go 泛型约束
 
5.1 什么是泛型约束
 
泛型约束用于限制类型参数的范围，规定  T  可以是什么类型、不能是什么类型，保证编译安全、代码可用。
 
没有约束 → 只能做赋值、打印等基础操作
有约束 → 可以做运算、调用方法、比较等特定操作
 
 
 
5.2 系统内置约束（最常用）
 
5.2.1  any 
 
代表任意类型，等价于  interface{} 。
 
go  
func Print[T any](v T) {
    fmt.Println(v)
}
 
 
适用：所有类型，但不能比较、不能运算。
 
 
 
5.2.2  comparable 
 
代表可使用 == / != 比较的类型。
map 的 key 必须用这个约束。
 
go  
func Contains[T comparable](list []T, target T) bool {
    for _, v := range list {
        if v == target {
            return true
        }
    }
    return false
}
 
 
适用：int、string、bool、指针、channel、数组等。
 
 
 
5.3 标准库约束（constraints）
 
需要导入：
 
go  
import "golang.org/x/exp/constraints"
 
 
常用：
 
-  constraints.Ordered ：可比较大小（int/float/string）
-  constraints.Integer ：所有整数
-  constraints.Float ：所有浮点数
 
示例：
 
go  
func Max[T constraints.Ordered](a, b T) T {
    if a > b {
        return a
    }
    return b
}
 
 
 
 
5.4 自定义联合类型约束
 
用  |  列出允许的类型。
 
go  
type Number interface {
    int | int64 | float64
}

func Add[T Number](a, b T) T {
    return a + b
}
 
 
适用：只允许特定几种类型。
 
 
 
5.5 方法接口约束（最强大）
 
要求类型必须实现某个方法。
 
go  
type Stringer interface {
    String() string
}

func PrintStr[T Stringer](v T) {
    fmt.Println(v.String())
}
 
 
适用：需要调用方法的场景。
 
 
 
5.6 类型约束的组合（多约束）
 
一个类型参数可以同时满足：
 
- 类型范围
- 方法实现
 
语法：
 
go  
type MyConstraint interface {
    int | string
    String() string
}
 
 
 
 
5.7 常见约束使用场景
 
约束 用途 
 any  通用容器、存储、打印 
 comparable  去重、查找、map key 
 Ordered  比较大小、排序 
自定义联合类型 限定数字/字符串 
方法接口 必须实现某个方法 
 
 
 
5.8 总结
 
- 约束 = 给泛型  T  设定允许范围
- 内置： any 、 comparable 
- 标准库： Ordered 、 Integer 、 Float 
- 自定义： 类型列表  或  方法接口 
- 约束越强，泛型函数能做的操作越多

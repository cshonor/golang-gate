# 06 - 新特性最佳实践

## 1. 先用标准库，再造轮子

- slice 操作：先看 `slices`
- map 操作：先看 `maps`
- 多错误汇总：先看 `errors.Join`

## 2. 代码风格建议

- 用标准库函数表达意图（可读性更强）
- 避免“为了炫技”引入新抽象（团队可维护性更重要）

## 3. 和你现有体系的衔接

- `01-datastruct/`：理解底层 → `06-go_new_features/`：用现代写法  
- `03-error_handling/`：理解错误链 → `errors.Join` 解决“多错误汇总”  
- `09-concurrency_patterns/`：并发任务汇总错误 → `errors.Join` 直接契合

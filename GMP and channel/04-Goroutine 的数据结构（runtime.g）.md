# Goroutine 的数据结构（runtime.g）

Goroutine 在运行时里对应结构体 **`g`**（源码中小写），日常说的 **G** 就是它：协程的「身份证 + 运行档案」。

## 1. 核心字段（理解调度必记）

### 1. `stack` — 栈

- 协程私有栈：局部变量、调用链。
- 初始很小，可按需扩容（连续栈）。

### 2. `gopc` / `pc` / `sp`（与 `sched` 配合）

- **`pc`**：程序计数器，下一条指令。
- **`sp`**：栈指针。
- 协程切换：**切出**时保存现场，**切入**时恢复，才能从断点继续执行。

### 3. `atomicstatus`（状态）

常见取值（名称以源码为准，随版本略有调整）：

| 状态 | 含义（直觉） |
|------|----------------|
| `_Gidle` | 空闲 |
| `_Grunnable` | 可运行，在队列里等 M |
| `_Grunning` | 正在某 M 上执行 |
| `_Gwaiting` | 等待（阻塞） |
| `_Gsyscall` | 在系统调用中 |
| `_Gdead` | 结束，待回收 |

### 4. `m`

- 当前执行该 G 的 **M**；谁在线程上跑我，就指向谁。

### 5. `sched`（`gobuf`）

- 保存被抢占/切换时的 **寄存器现场**（含 pc、sp 等）。
- 与栈信息一起，保证换回来还能继续跑。

### 6. `waitreason`

- 阻塞原因：**channel**、**锁**、**sleep**、**GC** 等，便于诊断与调度。

### 7. 抢占相关（如 `preempt` 等字段）

- Go 1.14+ **异步抢占**等机制会用到；运行过久可能被强制让出。

## 2. 一句话总结

**`g`** 保存协程的栈、状态、寄存器现场、绑定的 M、阻塞原因等，是 **GMP 调度、切换、阻塞与唤醒** 的数据载体。

| 记什么 | 记一句 |
|--------|--------|
| `g` 是啥 | 协程在 runtime 里的结构体 |

| 为啥阻塞 | `waitreason` |

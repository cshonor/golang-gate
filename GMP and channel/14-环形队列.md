环形队列（Circular Queue）核心总结
 
适用于 Go Channel 缓冲区等场景，可直接写入 Markdown 文档
 
 
 
一、核心定义
 
环形队列（也叫循环队列、环形缓冲区）是基于固定大小数组实现的 FIFO（先进先出）队列，通过「取模运算」将数组首尾逻辑相连，彻底解决普通顺序队列的假溢出问题，最大化利用数组空间，是 Go Channel 有缓冲模式的核心底层结构。
 
 
 
二、核心结构（Go Channel 对应实现）
 
字段/指针 作用 
 buf  存储元素的底层数组（固定长度，对应 Channel 缓冲区容量  dataqsiz ） 
 sendx （队尾指针） 下一个元素**入队（写入）**的下标 
 recvx （队头指针） 下一个元素**出队（读取）**的下标 
 qcount  队列中当前元素的数量（用于判断空/满状态） 
 dataqsiz  队列总容量（数组长度） 
 
 
 
三、核心原理与关键操作
 
1. 空/满状态判断（核心设计）
 
环形队列通过计数法解决「空/满状态重合」的经典问题：
 
- 队空： qcount == 0 （无元素）
- 队满： qcount == dataqsiz （元素占满数组）
- 下标更新逻辑（入队/出队）：
go  
// 入队：写入sendx位置，更新指针
sendx = (sendx + 1) % dataqsiz
qcount++

// 出队：读取recvx位置，更新指针
recvx = (recvx + 1) % dataqsiz
qcount--
 
 
取模运算  % dataqsiz  实现「指针走到数组末尾后自动绕回开头」的环形逻辑，严格保证 FIFO 顺序。
 
2. 入队（发送）流程
 
1. 校验队列未满（ qcount < dataqsiz ）
2. 将数据写入  buf[sendx]  位置
3. 更新  sendx  指针与  qcount  计数
4. 若有等待的接收方，唤醒对应 Goroutine（Go Channel 场景）
 
3. 出队（接收）流程
 
1. 校验队列非空（ qcount > 0 ）
2. 从  buf[recvx]  位置读取数据
3. 更新  recvx  指针与  qcount  计数
4. 若有等待的发送方，唤醒对应 Goroutine（Go Channel 场景）
 
 
 
四、核心优势
 
- 空间利用率 100%：无假溢出，数组空间完全利用
- 高性能：入队/出队均为 O(1) 常数时间操作
- 顺序保证：严格遵循 FIFO 语义，符合消息队列、Channel 的核心需求
- 缓存友好：连续内存布局，CPU 缓存命中率高
 
 
 
五、Go Channel 中的应用
 
- 有缓冲 Channel（ make(chan T, size) ）的缓冲区，本质就是一个带互斥锁保护的环形队列
- 无缓冲 Channel（ make(chan T) ）无环形队列，直接走「发送方-接收方同步交换」逻辑
- 环形队列配合  sendq/recvq  等待队列、互斥锁，实现了 Channel 的阻塞/唤醒机制
 
 
 
六、关键注意事项
 
1. 空/满判断：禁止用  sendx == recvx  判断，必须用  qcount  计数，避免逻辑错误
2. 并发安全：环形队列本身非线程安全，必须加互斥锁（Go Channel 内置  mutex  保证）
3. 内存泄漏：出队后需清空原位置元素，避免大对象无法被 GC 回收
4. 容量设计：根据业务峰值设置合理容量，避免过大浪费内存、过小失去缓冲意义
 
 
 
七、与其他队列的对比
 
特性 环形队列（数组实现） 普通顺序队列 链表队列 
空间利用率 100%（无假溢出） 低（假溢出浪费） 动态分配，无浪费 
时间复杂度 入队/出队 O(1) 入队 O(1)，出队 O(n) 入队/出队 O(1) 
内存连续性 连续，缓存友好 连续 非连续，缓存不友好 
适用场景 Go Channel、高性能固定大小缓存 简单小数据量场景 动态大小、元素数量不确定 
 
 
 
八、Go 简易实现（可直接复用）
 
go  
package main

import (
	"errors"
	"sync"
)

// RingBuffer 环形队列（参考 Go Channel 实现）
type RingBuffer[T any] struct {
	mu       sync.Mutex
	buf      []T
	sendx    uint
	recvx    uint
	qcount   uint
	capacity uint
}

// NewRingBuffer 创建指定容量的环形队列
func NewRingBuffer[T any](capacity uint) *RingBuffer[T] {
	if capacity == 0 {
		panic("capacity must be greater than 0")
	}
	return &RingBuffer[T]{
		buf:      make([]T, capacity),
		capacity: capacity,
	}
}

// Enqueue 入队，队列满时返回错误
func (r *RingBuffer[T]) Enqueue(item T) error {
	r.mu.Lock()
	defer r.mu.Unlock()

	if r.qcount == r.capacity {
		return errors.New("queue is full")
	}

	r.buf[r.sendx] = item
	r.sendx = (r.sendx + 1) % r.capacity
	r.qcount++
	return nil
}

// Dequeue 出队，队列空时返回错误
func (r *RingBuffer[T]) Dequeue() (T, error) {
	r.mu.Lock()
	defer r.mu.Unlock()

	var zero T
	if r.qcount == 0 {
		return zero, errors.New("queue is empty")
	}

	item := r.buf[r.recvx]
	r.buf[r.recvx] = zero // 清空避免内存泄漏
	r.recvx = (r.recvx + 1) % r.capacity
	r.qcount--
	return item, nil
}

// Len 返回队列当前元素数量
func (r *RingBuffer[T]) Len() uint {
	r.mu.Lock()
	defer r.mu.Unlock()
	return r.qcount
}

// IsEmpty 判断队列是否为空
func (r *RingBuffer[T]) IsEmpty() bool {
	r.mu.Lock()
	defer r.mu.Unlock()
	return r.qcount == 0
}

// IsFull 判断队列是否已满
func (r *RingBuffer[T]) IsFull() bool {
	r.mu.Lock()
	defer r.mu.Unlock()
	return r.qcount == r.capacity
}

# Go 数值类型（权威完整版）

## 一、整数类型

### 1. 有符号整数

| 类型 | 比特数 | 取值范围 | 适用场景 |
|---|---|---|---|
| **int8** | 8位 | -128 ~ 127 | 小范围整数、内存敏感场景 |
| **int16** | 16位 | -32768 ~ 32767 | 跨平台协议、短整型存储 |
| **int32** | 32位 | -2147483648 ~ 2147483647 | 网络协议、ID 标识、常规业务计算 |
| **int64** | 64位 | -9223372036854775808 ~ 9223372036854775807 | 大数计算、时间戳、金融金额 |
| **int** | 平台相关 | 32位系统同 int32；64位系统同 int64 | 业务开发默认类型，兼顾效率 |

### 2. 无符号整数

| 类型 | 比特数 | 取值范围 | 别名/用途 |
|---|---|---|---|
| **uint8** | 8位 | 0 ~ 255 | **byte**，字节流、二进制数据 |
| **uint16** | 16位 | 0 ~ 65535 | 跨平台网络协议、端口号 |
| **uint32** | 32位 | 0 ~ 4294967295 | 大基数计数、哈希容量 |
| **uint64** | 64位 | 0 ~ 18446744073709551615 | 超大计数、文件偏移量 |
| **uint** | 平台相关 | 同 int 平台特性 | 非负整数场景 |
| **uintptr** | 平台相关 | 同 uint | 指针地址表示（更偏底层） |

### 3. 关键说明

- 字面量支持：十进制、二进制（`0b/0B`）、八进制（`0o/0O`）、十六进制（`0x/0X`）
- 跨平台注意：**int/uint** 长度随系统变化；网络传输、存储序列化应使用固定长度类型（如 `int32/int64`）

## 二、浮点类型

| 类型 | 比特数 | 字节数 | 取值范围 | 精度 | 适用场景 |
|---|---|---|---|---|---|
| **float32** | 32位 | 4字节 | ±1.4e-45 ~ ±3.4e38 | 约 6-7 位有效数字 | 图形渲染、实时计算、内存优先 |
| **float64** | 64位 | 8字节 | ±4.9e-324 ~ ±1.8e308 | 约 15-16 位有效数字 | **业务默认**、科学计算 |

### 关键说明

- 字面量：支持十进制（如 `3.14`、`1e9`）、十六进制浮点（如 `0x1.2p-3`）
- 零值：浮点型默认零值为 `0.0`

## 三、复数类型

| 类型 | 比特数 | 组成 | 适用场景 |
|---|---|---|---|
| **complex64** | 64位 | float32（实部） + float32（虚部） | 信号处理、图像处理、数学运算 |
| **complex128** | 128位 | float64（实部） + float64（虚部） | 高精度科学计算、复数分析 |

### 关键说明

- 业务开发中使用较少，科学计算场景更常见
- 示例：`c1 := complex(1.5, 2.5)`；取实部/虚部：`real(c1)`、`imag(c1)`

## 四、典型声明与使用

### 1. 基础声明

```go
// 整数
var a int8 = 10
b := int16(20)           // 短声明
c := uint32(3000000)     // 无符号
d := int(42)             // 平台默认 int

// 浮点
var e float32 = 3.14
f := 6.28                // 默认为 float64
g := 1e-3                // 科学计数法（0.001）

// 复数
h := complex(1.5, 2.5)   // complex128
i := complex64(3 + 4i)   // 显式转换

_, _, _, _, _, _, _, _, _ = a, b, c, d, e, f, g, h, i
```

### 2. 进制表示示例

```go
bin := 0b1010   // 二进制，等于 10
oct := 0o12     // 八进制，等于 10
hex := 0xA      // 十六进制，等于 10

_, _, _ = bin, oct, hex
```

## 五、核心注意事项

1. **类型安全**：不同数值类型不支持隐式转换，必须显式转换（如 `int64(myInt32)`）
2. **零值特性**：整数/无符号为 `0`，浮点为 `0.0`，复数为 `(0+0i)`
3. **溢出风险**：固定长度整数（如 `int8`）超出范围会溢出，业务优先用 `int/int64`
4. **精度取舍**：`float32` 更省内存，`float64` 满足多数业务精度需求；金融场景通常配合 decimal 库

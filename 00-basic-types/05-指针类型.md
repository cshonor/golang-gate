# Go 指针类型 `*T`（完整版）

## 1. 指针是什么

- 存储**另一个变量在内存中的地址**（口语：指向谁）。
- 类型写法：`*T`，读作「指向 `T` 的指针」。
- **指针变量本身也是一个值**（这个值的内容通常是地址）；它自己也占内存，也可以再被取地址（`&&a` 少见，但合法）。
- **零值是 `nil`**：`var p *int` 不会随机指向某块内存。

## 2. 核心操作

- **`&x`**：取 `x` 的地址 → 得到 `*T`。
- **`*p`**：解引用 → 读写 `p` 指向的那份 `T`。

```go
a := 10
p := &a   // p 的类型是 *int
fmt.Println(*p) // 10
```

## 3. Go 指针的「安全」到底指什么（别神话，也别低估）

- **不支持指针算术**：像 `p++` 这种按地址步进，**直接非法**（编译期拒绝）。
- **不能随意把指针当别的类型乱转**（正常代码路径下更安全；真要强转走 `unsafe.Pointer` 另说）。
- **`nil` 指针解引用会 `panic`**：不是「随机改内存」那种 C 式未定义行为，但**照样会崩**，所以要先保证指针有效。

和 C 的常见对比（教学口径）：

- **未初始化的 `*T` 变量是 `nil`**，不会像野指针那样指向随机地址。
- **一般业务 Go 代码**里由 **GC** 管理对象生命周期，**不会像手动 `free` 那样典型地出现「悬垂引用 + 继续解引用」的 C 经典坑**；但一旦涉及 **`unsafe` / `cgo` / 最终化（finalizer）等**，仍然要自己负责生命周期与并发安全。

## 4. 指针最常见的 3 类用途

1. **在函数内修改调用方的变量**（把地址传进去）。
2. **避免大结构体反复值拷贝**（传 `*BigStruct`，按需求权衡可读性与性能）。
3. **实现经典数据结构**（链表、树：`next *Node` 等）。

最经典例子：**函数内修改外部变量**

```go
func set(p *int) {
	*p = 100
}

func main() {
	a := 1
	set(&a)
	// a == 100
}
```

## 5. 与 Java 的直觉对照

- **Java 的对象引用**：更像「你看不见细节的安全指针」，不能取地址、不能指针运算。
- **Go 的 `*T`**：**显式**的指针类型，能取地址、能出现 `nil`、能按值传递指针本身。

一句合并理解：**两者都可以表达「指向某块数据，而不是把整块数据到处拷贝」**；差别主要在**显式程度**与**语言规则**。

---

## 6. 三个最容易混淆的点（必看）

### ①「指针」vs「引用」别硬套名词

- Go 里更干净的说法是：**指针类型 `*T` 是一种类型**；变量里存的是**地址值**（或 `nil`）。
- 口语对比 Java/Python：**Java/Python 的引用通常不能 `&` 取底层地址**；Go 可以显式 `&`，规则更“底层可见”。

### ② `*T` 变量赋值：拷贝的是「地址这个值」

```go
p1 := &a
p2 := p1 // 拷贝指针变量保存的地址；p1、p2 指向同一份 a
```

所以：**指针变量按赋值/传参规则走，拷贝的是指针值本身**；被指向的那份数据是否“共享”，取决于你有没有多个指针指向它。

### ③ `nil` 指针不能解引用

```go
var p *int
// *p = 10 // panic：对 nil 解引用
```

要先让指针指向合法对象：`new(T)`、`&x`、构造函数返回的有效指针等。

---

## 7. 一句话总结

**Go 指针 = 存地址的值：`&` 取地址，`*` 读写指向对象；没有 C 式指针运算；`nil` 要小心；常用于「修改外部状态 / 少拷贝 / 建链接结构」。**

---

## 8. 附：值语义 vs 指针语义速查（防懵用）

> 下面用「**赋值后是否容易共享同一份底层数据**」做直觉分组（不是严格的官方术语表）。

| 类型（举例） | 变量里 roughly 存什么 | 赋值/传参的常见直觉 |
|---|---|---|
| `int`、`bool`、`float`… | 标量值本身 | 互相独立 |
| `*T` | 地址（或 `nil`） | 拷贝地址；多个指针可指向同一对象 |
| `string` | `string` 值（小头部） | 拷贝头部；底层字节只读共享 |
| `[N]T` 数组 | 整个数组值 | 拷贝整个数组（元素再按规则拷贝） |
| `struct` 值 | 结构体整体值 | 字段逐个拷贝；含指针/slice 等字段时仍可能出现共享 |
| `slice` | slice 头（ptr/len/cap） | 拷贝头；底层数组可共享 |
| `map` / `chan` | 运行时句柄式值 | 通常表现为共享同一张 map / 同一 channel |

如果你后面学到 `interface{}` / `any`、反射、`unsafe`，指针与生命周期的话题会继续加深；先把 **`nil`、`&`、`*`、共享与拷贝** 四条线捋直就够用了。

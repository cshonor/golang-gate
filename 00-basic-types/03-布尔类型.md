# Go bool 布尔类型（深度解读）

## 1. 基础定义

- 布尔类型 `bool` **只有两个合法值**：`true`、`false`
- **零值**：声明后默认是 **`false`**（这是 bool 最重要的语言特性之一）
- **内存大小**：Go 规范**不保证** `bool` 固定占 1 字节；常见实现里它很小，工程上可用 `unsafe.Sizeof(bool(0))` 在你当前平台自证

## 2. 基础声明方式

```go
// 标准声明
var ok bool
ok = true

// 短变量声明
ok2 := false

// 直接初始化
var isLogin = true
```

## 3. 核心语言特性（Go 强类型最典型的体现）

### 绝对禁止隐式类型转换

- **布尔值与整数 `0/1`、数字之间不能自动互转**
- 典型错误（编译失败）：

```go
// if 1 {}           // 不能把整数直接当条件
// var b bool = 1    // 不能把数字赋给 bool
```

- 正确写法：把“数字条件”写成显式比较：

```go
num := 1
if num != 0 {
	// ...
}
```

### 支持的逻辑运算符

| 运算符 | 含义 |
|--------|------|
| `&&` | 逻辑与 |
| `||` | 逻辑或 |
| `!` | 逻辑非 |

### 短路求值

- `a && b`：`a` 为 `false` 时，`b` **不会求值**
- `a || b`：`a` 为 `true` 时，`b` **不会求值**

## 4. 官方/工程最佳实践

1. **命名**
   - 状态类：`isXXX`、`hasXXX`、`canXXX`
   - 通用标记：`ok`（标准库里非常常见）
2. **不要用 `1/0` 代替 bool** 存业务状态
3. **条件判断直接用 bool**，避免冗余：

```go
if isOk {
}

// 冗余（可读性差）
// if isOk == true {}
```

## 5. 易错补充

- `bool` 是**值类型**：赋值会拷贝值本身（很小）
- `bool` **支持 `==` / `!=`**，因此**可以作为 `map` 的 key**
- `bool` **没有 `nil`**：只有 `true/false`，零值永远是 `false`

# Go bool 布尔类型（深度解读）

## 1. 基础定义

- 布尔类型 `bool` **只有两个合法值**：`true`、`false`
- **零值**：声明后默认是 **`false`**（这是 bool 最重要的语言特性之一）
- **内存大小**：Go 规范**不保证** `bool` 固定占 1 字节；常见实现里它很小，工程上可用 `unsafe.Sizeof(bool(0))` 在你当前平台自证

## 2. 基础声明方式

```go
// 标准声明
var ok bool
ok = true

// 短变量声明
ok2 := false

// 直接初始化
var isLogin = true
```

## 3. 核心语言特性（Go 强类型最典型的体现）

### 绝对禁止隐式类型转换

- **布尔值与整数 `0/1`、数字之间不能自动互转**
- 典型错误（编译失败）：

```go
// if 1 {}           // 不能把整数直接当条件
// var b bool = 1    // 不能把数字赋给 bool
```

- 正确写法：把“数字条件”写成显式比较：

```go
num := 1
if num != 0 {
	// ...
}
```

### 支持的逻辑运算符

| 运算符 | 含义 |
|--------|------|
| `&&` | 逻辑与 |
| `||` | 逻辑或 |
| `!` | 逻辑非 |

### 短路求值

- `a && b`：`a` 为 `false` 时，`b` **不会求值**
- `a || b`：`a` 为 `true` 时，`b` **不会求值**

## 4. 官方/工程最佳实践

1. **命名**
   - 状态类：`isXXX`、`hasXXX`、`canXXX`
   - 通用标记：`ok`（标准库里非常常见）
2. **不要用 `1/0` 代替 bool** 存业务状态
3. **条件判断直接用 bool**，避免冗余：

```go
if isOk {
}

// 冗余（可读性差）
// if isOk == true {}
```

## 5. 易错补充

- `bool` 是**值类型**：赋值会拷贝值本身（很小）
- `bool` **支持 `==` / `!=`**，因此**可以作为 `map` 的 key**
- `bool` **没有 `nil`**：只有 `true/false`，零值永远是 `false`
- 关于「浅拷贝 / 深拷贝」这类说法怎么套在 `bool` 上：见 **第 6 节**

## 6. `bool` 与「浅拷贝 / 深拷贝」（先给结论）

**`bool` 这种最基础的标量值类型，不存在「浅拷贝 vs 深拷贝」的区分：赋值/传参永远是按 `bool` 规则做一份独立的值拷贝**，没有「只拷贝指针、底层共享」这种歧义空间。

### 6.1 什么叫「完整值拷贝」？

当你赋值、传参时，`true/false` 会按语言规则产生**互相独立**的副本；改一个不会影响另一个：

```go
var a bool = true
b := a

b = false
// a 仍是 true；b 是 false
```

### 6.2 那什么时候才会聊「浅 / 深」？

工程上大家说的「浅拷贝」，通常指：**拷贝了某个“把柄”（例如切片头、map 头、指针值），新旧变量仍然指向同一份可变底层数据**。

典型例子（切片赋值拷贝的是切片头，底层数组共享）：

```go
s1 := []int{1, 2, 3}
s2 := s1
s2[0] = 999
// s1 也会变成 [999 2 3]
```

「深拷贝」通常指：**把底层数据也复制一份，让新旧变量彻底分家**（例如对切片 `append(nil, s...)` / `copy` 到新底层数组，或自己递归复制结构体里指针指向的内容——具体怎么做取决于你的类型）。

### 6.3 回到 `bool`：为什么几乎没人谈深浅？

1. `bool` 没有「额外底层可变数据」需要共享：它表达的就是这个标量真值本身  
2. 赋值/传参按语言语义就是**独立的 `bool` 值**（不是拷贝某个指针去共用一个缓冲区）  
3. 因此口语里会说：**它就是值拷贝；用“深/浅”去分类它反而容易绕晕**

### 6.4 一句话对照表（注意别扩大化）

| 场景 | 拷贝行为（直觉） | 还纠结「深/浅」吗？ |
|---|---|---|
| `bool` / 整数 / 浮点 等标量 | 独立标量值 | 一般**不纠结**（没有共享底层那一层） |
| 数组 `[N]T` | 拷贝整个数组值（元素按赋值规则逐个拷贝） | 一般**不纠结**为深浅；但若 `T` 含指针/切片等，元素里仍可能出现**别名** |
| `struct` 值 | 拷贝整个结构体值（字段按赋值规则逐个拷贝） | 字段里若有指针/slice/map，**仍可能出现共享底层**；这不是 `bool` 那种「纯标量」模型 |
| `string` | 拷贝 `string` 值（小头部）；底层字节**只读共享** | 不等同于「把字符串所有字节再复制一份」 |
| `slice` / `map` / `chan` / 指针 | 常拷贝的是「头/指针值」，默认可变底层共享 | 这里最常说**浅拷贝**；需要分家时再谈**深拷贝**怎么做 |

### 6.5 可运行对比：`bool` vs `slice`

```go
package main

import "fmt"

func main() {
	var a bool = true
	b := a
	b = false
	fmt.Println("bool:", a, b)

	s1 := []int{1, 2, 3}
	s2 := s1
	s2[0] = 999
	fmt.Println("slice:", s1, s2)
}
```

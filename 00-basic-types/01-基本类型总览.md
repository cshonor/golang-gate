# Go 基本类型总览（完整版）

## 一、Go 类型体系两大分类

### 1. 基础类型（值类型）

- **数值类型**
  - 有符号：`int`、`int8`、`int16`、`int32`、`int64`
  - 无符号：`uint`、`uint8`、`uint16`、`uint32`、`uint64`、`uintptr`
  - 浮点：`float32`、`float64`
  - 复数：`complex64`、`complex128`
  - 别名：`byte`(`uint8`)、`rune`(`int32`)
- **布尔类型**：`bool`（仅 `true/false`）
- **字符串类型**：`string`（不可变）
- **指针类型**：`*T`（存储地址，指向 `T`）

### 2. 复合类型

- 数组：`[n]T`（固定长度）
- 切片：`[]T`（动态数组视图）
- Map：`map[K]V`（键值哈希）
- 结构体：`struct`（字段聚合）
- 函数：`func`
- 通道：`chan T`（goroutine 通信）
- 接口：`interface{}`

---

## 二、Go 类型核心特点

1. **赋值语义是“值复制”**  
   - 数组、结构体：复制全部字段/元素
   - 切片、map、chan、func、指针：复制的是描述符/引用值，底层数据可能共享
2. **强类型、无隐式转换**  
   不同类型不能自动混用，必须显式转换
3. **声明即零值初始化**  
   - 数值：`0`
   - bool：`false`
   - string：`""`
   - 指针/切片/map/chan/func/interface：`nil`
4. **类型边界清晰、安全**  
   无隐式提升、无自动装箱拆箱，减少隐式类型坑

## 三、Go ↔ Java 类型对应对照

| Go 类型 | Java 类型 | 关键差异 |
|---|---|---|
| `bool` | `boolean` | 语义接近 |
| `int` / `int64` | `int` / `long` | Go `int` 随平台 32/64 位变化 |
| `uint` 系列 | 无 | Java 原生无无符号整型 |
| `float64` | `double` | Go 业务里也常默认 `float64` |
| `string` | `String` | Go string 是不可变字节序列；Java 有常量池等实现差异 |
| `*T` | 对象引用 | Go 指针更接近底层地址模型，但不支持通用指针算术 |

---

## 补充易错区分

- 数组是值语义；切片是轻量描述符，底层数组可能共享  
- struct 是值语义；`*struct` 常呈现“引用效果”  
- Go 没有类继承体系，主要通过接口实现多态

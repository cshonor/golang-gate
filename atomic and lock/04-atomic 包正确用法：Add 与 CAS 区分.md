# atomic 包正确用法：Add 与 CAS 区分

> 搞懂 `atomic`：**简单计数/读写** 用 Add、Load、Store；**带条件的修改** 才考虑 CAS，且要有明确失败语义，不要无脑死循环自旋。

## 1. Add / Load / Store / Swap

- **`Add`**：原子加减，适合**计数器、引用计数、序号**等，语义是「一定成功」（在合法范围内）。
- **`Load` / `Store`**：原子读、写单个字长变量，保证可见性与原子性，不涉及「比较」。
- **`Swap`**：原子写入并返回旧值，也是完整原子操作。

**场景**：统计 QPS、无锁计数、标志位简单翻转（若逻辑允许）。

## 2. CAS（CompareAndSwap）

- 语义：**仅当当前值 == 期望值时**才写入新值，否则失败并告诉你当前值（或只返回 false）。
- **可能失败**：并发下别人先改了，你就要决定——重试策略、还是放弃、还是走别的路径。
- **不要**把「无限 `for { CAS }`」当默认写法（除非极少数无锁算法且已证明无饥饿/有退避）。

**场景**：状态机迁移（如 idle → running）、只初始化一次的无锁路径、无锁数据结构中的节点链接。

## 3. 万能口诀（背）

> **锁一段代码（多语句、多变量不变量）→ 用 `Mutex`。  
> 锁一个变量（单字长、简单原子语义）→ 用 `atomic`。**

## 4. Add 与 CAS 怎么选

| 需求 | 倾向 |
|------|------|
| 单纯 +1 / -1 计数 | `Add` |
| 只读最新值 | `Load` |
| 无条件写入 | `Store`（注意与业务语义是否允许） |
| 「仅当为旧值 A 才改成 B」 | `CompareAndSwap` |

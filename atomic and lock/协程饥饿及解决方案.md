# 协程饥饿及解决方案

> 课表：**锁实现原理 · 基础版** — 协程饥饿及解决方案

## 1. 什么是饥饿（starvation）

- 某些执行者 **长期拿不到锁**（或得不到调度），而其他人一直成功 → **不公平**、延迟不可控。

## 2. 常见原因

- **锁实现不公平**：唤醒顺序固定、新来者插队（barging）。  
- **高优先级/热点 goroutine** 挤占（调度与锁叠加）。  
- **读写锁**：大量读导致写者饿死（若实现不防饥饿）。

## 3. 典型解决思路

| 思路 | 说明 |
|------|------|
| **队列 FIFO** | 按等待顺序授予锁（实现成本更高） |
| **防插队 / 混合模式** | 一段时间自旋后优先保证等待最久者（如 Go mutex **饥饿模式** 思想） |
| **带时限** | `TryLock`、超时重试（业务层） |
| **缩小临界区** | 减少「占着锁不放」的时间 |

## 4. Go 相关（概念）

- `sync.Mutex`：**正常模式** 与 **饥饿模式** 切换，缓解长尾饥饿（细节以源码/官方文档为准）。

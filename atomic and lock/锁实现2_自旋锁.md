# 锁实现 2：自旋锁（spin lock）

> 课表：**锁实现原理 · 基础版** — 自旋锁

## 1. 行为

- 获取锁失败时 **不睡眠**，在 CPU 上 **循环重试**（常配合 `atomic` / CAS）。  

- 核心逻辑：获取锁失败时不睡眠、不阻塞，在CPU上循环重试（常配合 atomic/CAS 实现）
​
- 优势：锁持有时间极短时，可避免线程阻塞、上下文切换的开销，性能远高于阻塞锁
​
- 本质：用「CPU空转」换「无阻塞」，是典型的空间换时间优化

## 2. 典型实现直觉

// 伪代码：Go 自旋锁核心实现

for !atomic.CompareAndSwapUint32(&lock, 0, 1) {

    // 自旋优化：空转或PAUSE指令，降低CPU功耗、减少流水线冲刷

    runtime.Gosched() // 或 CPU PAUSE 指令（如 x86 的 pause）

}

// 释放锁：原子写回0，需保证内存序正确
atomic.StoreUint32(&lock, 0)


- **释放**：`store(lock, 0)` 或原子写回（需与架构内存序一致）。

- 关键细节：自旋时不能死循环空转，必须加 PAUSE / Gosched() ，否则会占满CPU、影响调度

​
- 内存序要求：释放锁的写操作必须带正确的内存屏障，保证临界区代码对其他线程可见

## 3. 适用场景

- 临界区 **极短**、竞争 **不极端**（内核短路径、部分 runtime 内部）。  
- 用户态业务层长时间自旋通常 **不划算**。

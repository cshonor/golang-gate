# 自旋锁的代价

> 课表：**锁实现原理 · 基础版** — 自旋锁的代价

## 1. CPU 时间

CPU 时间：算力的直接浪费

核心问题

自旋锁的本质是忙等待（Busy Waiting）：线程获取锁失败时，不会主动让出 CPU，而是在循环中不断尝试获取锁，直到成功。

等待线程会完全占用 CPU 核心空转，无法执行其他有效任务，造成算力的直接浪费。
当大量线程同时竞争同一把自旋锁时，会出现 “多线程空转抢锁” 的局面，导致系统整体吞吐量断崖式下跌，甚至出现 CPU 占用率 100% 但业务无进展的情况。

补充原理

自旋锁的设计初衷是短时间锁竞争：当锁的持有时间极短（如几纳秒到微秒级），线程切换（上下文切换）的开销（约数微秒）远大于自旋等待的开销，此时自旋锁更高效。但一旦锁持有时间变长，自旋的代价会指数级上升。

## 2. 缓存与总线（cache line bouncing）

- 多核反复 CAS 同一 **cache line** → 缓存行在核间 **来回失效/同步**（false sharing 会加重）。  
- 高竞争下，自旋不仅是「费电」，还拖慢 **所有人**。

## 3. 调度与公平

- 自旋久不占 CPU 时间片可能饿死他人；或与 OS 调度策略交互产生 **延迟尖刺**。  
- 用户态无「队列保证」的朴素自旋还可能有 **不公平**。

## 4. 结论（承上启下）

- 纯自旋只适合 **短等待**；长了要 **让出 CPU** → **阻塞 + 唤醒**（下一节）。

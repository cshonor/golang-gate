# 03 - 自定义错误类型

前置：[02 - 错误基本语法](./02%20-%20错误基本语法.md)（`error` 接口、`errors.New`）。若错误会被 `fmt.Errorf("...: %w", err)` 包装，提取类型请用 [05 - errors.As](./05-errors.Is%20与%20errors.As%20使用.md)。

---

## 1. 一句话对比

| | 普通错误 | 自定义错误 |
|---|----------|------------|
| 常见写法 | `errors.New("...")` / `fmt.Errorf("...")` | 自定义结构体 + 实现 `Error() string` |
| 能带什么 | 基本只有**文案**（链路由 `%w` 扩展） | **字段**：错误码、业务字段名、ID 等 |

直觉对比：

- 仅字符串：「参数错误」  
- 带数据：`Code=400`，`Field=name`，`Msg=用户名不能为空` —— 便于映射 HTTP 状态码、结构化日志与监控。

---

## 2. 为什么要自定义？

`errors.New` 只能表达**固定字符串**，例如：

```go
return errors.New("参数错误")
```

业务里往往需要机器可读的 **Code**、给人看的 **Msg**、以及 **Field** / **ID** 等上下文。这些需要**结构体字段**承载，再让该类型实现 `error` 接口。

---

## 3. `error` 接口与 `Error()` 方法

```go
type error interface {
	Error() string
}
```

任意类型只要实现了 `Error() string`，返回值就可以当作 `error` 使用。

### 3.1 最小示例

```go
type BizError struct {
	Code int
	Msg  string
}

func (e *BizError) Error() string {
	return e.Msg
}
```

---

## 4. 返回与解析：类型断言与 `errors.As`

### 4.1 直接返回、未包装时：类型断言

```go
func login(name string) error {
	if name == "" {
		return &BizError{
			Code: 400,
			Msg:  "用户名不能为空",
		}
	}
	return nil
}

// 调用方
err := login("")
if myErr, ok := err.(*BizError); ok {
	fmt.Println("错误码：", myErr.Code)
	fmt.Println("错误信息：", myErr.Msg)
}
```

### 4.2 中间曾用 `%w` 包装时：用 `errors.As`

若链路中存在 `fmt.Errorf("login: %w", err)`，`*BizError` 可能被包在错误链里，此时 `err.(*BizError)` 可能失败。应使用 `errors.As`（见 [05](./05-errors.Is%20与%20errors.As%20使用.md)）。

```go
var be *BizError
if errors.As(err, &be) {
	fmt.Println(be.Code, be.Msg)
}
```

---

## 5. 带 Code、Field、Msg 的完整示例（可直接改项目用）

```go
package main

import (
	"errors"
	"fmt"
)

type BizError struct {
	Code  int
	Msg   string
	Field string // 可选：哪个字段不合法
}

func (e *BizError) Error() string {
	if e.Field != "" {
		return fmt.Sprintf("[%d] %s (field=%s)", e.Code, e.Msg, e.Field)
	}
	return fmt.Sprintf("[%d] %s", e.Code, e.Msg)
}

func validateName(name string) error {
	if name == "" {
		return &BizError{
			Code:  400,
			Msg:   "用户名不能为空",
			Field: "name",
		}
	}
	return nil
}

func main() {
	err := validateName("")
	var be *BizError
	if errors.As(err, &be) {
		fmt.Println("code:", be.Code, "msg:", be.Msg, "field:", be.Field)
	}
}
```

说明：`Field` 为空时，`Error()` 可只输出 code + msg，避免日志里出现无意义的 `field=`。

---

## 6. 这样做的好处

- **按类型分流**：参数类 / 权限类 / 系统类映射不同 HTTP 状态码或重试策略。  
- **结构化日志与告警**：把 `code`、`field` 打成 JSON 字段，比纯字符串更好检索（与 [08 - 错误与日志集成](./08%20-%20错误与日志集成.md) 配合）。  
- **与 `errors.Is` 配合**：可为包定义哨兵值；需要**字段**时再用自定义类型 + `As`（见 [05](./05-errors.Is%20与%20errors.As%20使用.md)）。

---

## 7. 速记

- 普通错误：多为**字符串**（或包装链）。  
- 自定义错误：**结构体 + `Error()`** → 仍是 `error`。  
- 调用方：**直接类型**用断言；**可能被包装**用 `errors.As`。

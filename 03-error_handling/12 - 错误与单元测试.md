# 12 - 错误与单元测试

测试里要验证的不仅是「有没有错」，而是 **是否是预期的那类错**（哨兵、包装链、自定义类型）。前置：[06](./06%20-%20errors.Is与errors.As使用.md)、[07](./07%20-%20Sentinel%20Error哨兵错误设计.md)。

---

## 1. 标准库：`errors.Is` / `errors.As`

```go
_, err := DoSomething()
if !errors.Is(err, ErrBadInput) {
	t.Fatalf("want ErrBadInput, got %v", err)
}
```

```go
var ve *ValidationError
if !errors.As(err, &ve) || ve.Field != "name" {
	t.Fatalf("unexpected: %v", err)
}
```

---

## 2. `testify` 常用断言

```go
import (
	"github.com/stretchr/testify/require"
)

require.Error(t, err)
require.ErrorIs(t, err, ErrNotFound)
```

`ErrorIs` 等价于对 `errors.Is` 的封装，适合表驱动测试里一行断言。

---

## 3. 表驱动测试错误分支

- 用字段 `wantErrIs`、`wantAsType` 描述期望，循环里 `errors.Is` / `As`。
- **不要**依赖 `err.Error()` 的完整字符串（除非测固定文案且明确不会国际化/包装）。

---

## 4. 覆盖率建议

- 对 **每个** `errors.Is` / `As` 分支写至少一条用例。
- 包装路径：`fmt.Errorf("...: %w", inner)` 后仍能 `ErrorIs` 到 `inner`。

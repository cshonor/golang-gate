# 02 - 泛型语法基础（新手向）

> **05-Go Generics**  
> 建议先读：[01-Introduce](./01-Introduce.md)（**为什么要有泛型**）。读完本篇再往下：[03-泛型函数](./03-泛型函数.md)。

如果你**完全没接触过** Go 泛型，可以只把下面三件事读明白，其它先跳过也没关系：

1. **方括号 `[T any]` 是在给「类型」留一个空位，名字叫 `T`。**
2. **调用时常常不用写类型，编译器会猜（类型推断）。**
3. **新手阶段先只记 `any`：表示「随便哪种类型都行，先能跑起来」。**

---

## 一、先用大白话：泛型到底在解决什么？

你想写一个函数：**不管是 `int` 还是 `string`，都想用同一套打印逻辑**。

### 没有泛型时（你可能见过这样写）

要么写两遍函数，要么用 `interface{}`（现在的 `any`）然后自己做类型判断——**容易啰嗦，也容易运行时才发现写错了类型**。

### 有泛型时（多一个「类型空位」）

你写**一份**函数，告诉编译器：「这里有一个类型，我暂时叫它 `T`，具体是 `int` 还是 `string`，看调用处再说。」

这就是泛型：**代码复用 + 尽量在编译期检查类型**。

---

## 二、第一例子：只记这一段就够入门

```go
package main

import "fmt"

// [T any] 读作：这里有一个类型参数 T，约束是 any（任意类型都行）
func Print[T any](v T) {
	fmt.Println(v)
}

func main() {
	Print(10)       // 编译器推断：这里的 T 是 int
	Print("hello")  // 编译器推断：这里的 T 是 string
}
```

你可以把 **`Print[T any]`** 理解成：

- **`Print`**：函数名，和以前一样。  
- **`[T any]`**：多出来的那一截，**专门给「类型」用的参数**。  
- **`T`**：你给这个「未知类型」起的名字（叫 `A`、`X` 也行，大家习惯用 `T`）。  
- **`any`**：约束，意思是 **`T` 可以是任意类型**（新手阶段先死记这句）。  
- **`(v T)`**：参数 `v` 的类型是 **`T`**——也就是说，`v` 的类型和你在方括号里选的那个类型一致。

**调用时**：你写 `Print(10)` 就行，**不用**写 `Print[int](10)` 也能编译——这叫**类型推断**。

---

## 三、方括号语法：慢慢拆一行

定义泛型函数时，模板长这样（先认形状，不必背中文术语）：

```text
func 名字 [ 类型参数名 约束 ] ( 普通参数 ) 返回值 { ... }
         ↑________________↑
              多出来的这一块
```

对应到上面的例子：

```text
func Print [ T any ] ( v T )
          ↑  ↑   ↑     ↑
          │  │   │     └── 参数 v 的类型就是 T
          │  │   └──────── 约束：任意类型
          │  └──────────── 类型参数的名字
          └─────────────────类型参数列表
```

---

## 四、什么时候要自己写 `[具体类型]`？（知道有这回事即可）

大多数时候**不用写**，编译器能从实参推断出来。

少数情况推断不出来或你想写清楚，可以显式写：

```go
Print[int](10)
```

新手阶段**遇到编译报错再学**这一手也完全来得及。

---

## 五、`any` 先够用：`comparable` 和 `cmp.Ordered` 是什么？（先记名字）

等你开始写 **`map` 的键**、或者要写**去重**、**比大小（`>`）** 时，才会碰到更细的约束。这里只给你**直觉**，不要求现在就掌握。

| 名字 | 一句话（先有个印象） |
|------|----------------------|
| **`any`** | 任意类型；上面 `Print` 已经够用。 |
| **`comparable`** | 能用 `==` / `!=` 的类型；**`map` 的键**必须是这种。 |
| **`cmp.Ordered`**（Go 1.21+） | 能写 `a > b` 这种比较大小的类型；要写 `Max` 之类才会用到。 |

**想进阶时**再看：[05-泛型类型约束基础](./05-泛型类型约束基础.md)（篇首有 **「新手先看：大白话五步」**，把 `any` / `comparable` / `cmp.Ordered` 一次说透）。

## 六、泛型结构体：把「盒子里的东西」也变成 `T`

你已经会 `Print[T any](v T)` 了，结构体同理：**字段类型也可以是 `T`**。

```go
// 一个盒子，里面装的东西类型叫 T
type Box[T any] struct {
	Content T
}

// 装整数
var a = Box[int]{Content: 42}

// 装字符串
var b = Box[string]{Content: "hi"}
```

读法：**`Box[int]`** = 「`T` 具体选成 `int` 的盒子」。

---

## 七、带泛型的「方法」（知道形状即可）

如果**类型本身**是 `Container[T]`，那它的方法名后面也要带上 `[T]`，表示「我还在用同一个 `T`」：

```go
type Container[T any] struct {
	value T
}

func (c *Container[T]) Set(v T) { c.value = v }
func (c Container[T]) Get() T  { return c.value }
```

更细的规则（比如「能不能再给方法单独加一套类型参数」）在 [07 - 泛型方法](./07-泛型方法.md)，**第一遍可以不看**。

---

## 八、新手最容易安心的三句话

1. **`[T any]` 只是多出来的「类型空位」，不是魔法。**  
2. **先会用 `Print[T any]` 和 `Box[T any]`，你就已经入门一半了。**  
3. **`comparable` / `cmp.Ordered` 不急**：写到 `map`、去重、`Max` 再回头看 [05](./05-泛型类型约束基础.md)。

## 九、附录：进阶示例代码（第一遍可跳过）

下面这些**第一遍不用读**；等你开始写 `map`、去重、`Max` 再回来对照即可。

### `comparable`：切片去重（需要 `map`，键要能 `==`）

```go
func Deduplicate[T comparable](slice []T) []T {
	seen := make(map[T]struct{})
	out := make([]T, 0, len(slice))
	for _, v := range slice {
		if _, ok := seen[v]; !ok {
			seen[v] = struct{}{}
			out = append(out, v)
		}
	}
	return out
}
```

### 只允许几种数字：`int | int64 | float64`

```go
type Number interface {
	int | int64 | float64
}

func Sum[T Number](a, b T) T {
	return a + b
}
```

### 比大小：`cmp.Ordered`（Go 1.21+）

```go
import "cmp"

func Max[T cmp.Ordered](a, b T) T {
	if a > b {
		return a
	}
	return b
}
```

### 小栈（字段是 `[]T`）

```go
type Stack[T any] struct {
	items []T
}

func (s *Stack[T]) Push(item T) { s.items = append(s.items, item) }

func (s *Stack[T]) Pop() (T, bool) {
	if len(s.items) == 0 {
		var zero T
		return zero, false
	}
	i := len(s.items) - 1
	v := s.items[i]
	s.items = s.items[:i]
	return v, true
}
```

## 复习速记（抄在小本本上）

| 写什么 | 什么意思 |
|--------|----------|
| `[T any]` | 类型空位叫 `T`，暂时啥类型都行 |
| `Print(10)` | 编译器猜 `T` 是 `int` |
| `Box[int]` | 把 `T` 定死成 `int` 的盒子 |

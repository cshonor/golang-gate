# 02 - 泛型语法基础（Go 版）

Go **1.18+** 支持泛型：用**类型参数**在**编译期**复用代码，配合**约束**限制 `T` 能做什么。

## 1. 核心三要素

- **类型参数**：`[T Constraint]` 等形式。
- **类型约束**：缩小 `T` 的范围（`any`、`comparable`、自定义接口等）。
- **泛型函数 / 泛型类型**：函数、结构体、接口等都可以带类型参数。

## 2. 泛型函数：长什么样

```go
// 定义：类型参数写在函数名后面
func 函数名[类型参数 约束](参数列表) 返回值 {
	// ...
}

// 调用：通常让编译器推断类型
函数名(实参)

// 需要时也可显式写类型实参
函数名[具体类型](实参)
```

### 示例：`cmp.Ordered`（Go 1.21+ 标准库）

比较大小时用 **`cmp.Ordered`**（可排序类型），无需 `golang.org/x/exp/constraints`：

```go
import "cmp"

func Max[T cmp.Ordered](a, b T) T {//a 和 b 必须是同一个类型 T
	if a > b {
		return a
	}
	return b
}
```

> 若你使用 **Go 1.18～1.20**，可改用 `golang.org/x/exp/constraints` 里的 `constraints.Ordered`，或自己写 `type Ordered interface { int | int64 | float64 | string }` 等联合类型。

### 任意类型：`any`

```go
func Print[T any](value T) {
	fmt.Println(value)
}
```

Go 语言泛型实现的切片去重函数
### `comparable`（可做 `==` / `!=`，如 map 的键）

```go
// 定义泛型函数 T 必须是可比较类型（comparable）
func Deduplicate[T comparable](slice []T) []T {
	// seen 用来记录已经出现过的元素，值用空结构体（不占内存）
	seen := make(map[T]struct{})

	// 预分配容量，避免多次扩容，提升性能
	out := make([]T, 0, len(slice))

	// 遍历原切片
	for _, v := range slice {
		// 如果元素没在 map 里 → 没出现过
		if _, ok := seen[v]; !ok {
			seen[v] = struct{}{}  // 标记为已出现
			out = append(out, v)  // 加入结果切片
		}
	}

	return out // 返回去重后的切片
}
```

### 自定义联合约束

```go
type Number interface {
	int | int64 | float64
}

func Sum[T Number](a, b T) T {
	return a + b
}
```

### 方法约束

```go
type Stringer interface {
	String() string
}

func Show[T Stringer](t T) {
	fmt.Println(t.String())
}
```

## 3. 泛型类型（结构体 / 切片 / map）

### 泛型结构体

```go
type Box[T any] struct {
	Content T
}

func main() {
	_ = Box[int]{Content: 100}
	_ = Box[string]{Content: "hello"}
}
```

### 泛型 map 类型（键要 `comparable`）

```go
type MyMap[K comparable, V any] struct {
	data map[K]V
}
```

## 4. 泛型方法（依附在泛型类型上）

方法接收者必须是**已带类型参数**的类型；详见 [07 - 泛型方法](./07-泛型方法.md)。

```go
type Container[T any] struct {
	value T
}

func (c *Container[T]) Set(v T) { c.value = v }
func (c Container[T]) Get() T   { return c.value }
```

## 5. 常用约束对照（概念）

| 约束 | 含义（直觉） |
|------|----------------|
| `any` | 任意类型 |
| `comparable` | 可 `==` / `!=`（map 键等） |
| `cmp.Ordered`（1.21+） | 可比较大小（`>` 等） |

展开与 `~`、联合、方法约束等见 [05 - 泛型类型约束基础](./05-泛型类型约束基础.md)、[06 - 泛型类型约束进阶](./06-泛型类型约束进阶.md)。

## 6. 几条重要规则

1. 需要 **Go 1.18+**。
2. **运行时没有「泛型类型」的反射信息**与 Java 泛型擦除类似，按具体实例类型来。
3. **map 的键**类型必须 `comparable`。
4. 泛型方法的类型参数来自**接收者类型**，一般不能凭空再给方法单独加一套全新类型参数（Go 1.18 设计如此）。
5. 例如不能写 `make([]T, n)` 里用类型参数当数组长度那种——长度必须是常量（老规则仍适用）。

### 完整小例子：栈

```go
type Stack[T any] struct {
	items []T
}

func (s *Stack[T]) Push(item T) {
	s.items = append(s.items, item)
}

func (s *Stack[T]) Pop() (T, bool) {
	if len(s.items) == 0 {
		var zero T
		return zero, false
	}
	i := len(s.items) - 1
	v := s.items[i]
	s.items = s.items[:i]
	return v, true
}
```

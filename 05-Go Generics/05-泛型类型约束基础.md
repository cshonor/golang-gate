# 05 - 泛型类型约束基础

**约束**用来限制类型参数 `T`「允许是哪些类型、能做什么操作」。没有约束时，`T` 往往只能做很有限的事（如传递、打印）；加上约束才能比较、运算、调方法等。**`~` 底层类型、复杂联合与组合约束**见 [06 - 泛型类型约束进阶](./06-泛型类型约束进阶.md)。

---

## 1. 内置常用约束

### `any`

任意类型，等价于 `interface{}`。

```go
func Print[T any](v T) {
	fmt.Println(v)
}
```

### `comparable`

可用 `==` / `!=` 的类型；**map 的键**必须满足 `comparable`。

```go
func Contains[T comparable](list []T, target T) bool {
	for _, v := range list {
		if v == target {
			return true
		}
	}
	return false
}
```

---

## 2. 标准库 `cmp.Ordered`（Go 1.21+）

需要**比较大小**（如 `>`）时，优先用 **`cmp.Ordered`**（**不要**误用 `comparable` 来写 `>`）。

```go
import "cmp"

func Max[T cmp.Ordered](a, b T) T {
	if a > b {
		return a
	}
	return b
}
```

> **Go 1.18～1.20** 可继续使用 `golang.org/x/exp/constraints` 中的 `constraints.Ordered`，或手写 `int | int64 | string` 等联合类型。

---

## 3. 联合类型：只允许几种具体类型

```go
type Number interface {
	int | int64 | float64
}

func Add[T Number](a, b T) T {
	return a + b
}
```

---

## 4. 方法约束

要求 `T` 必须实现某接口：

```go
type Stringer interface {
	String() string
}

func PrintStr[T Stringer](v T) {
	fmt.Println(v.String())
}
```

---

## 5. 组合约束与 `~`（提要）

「类型列表 + 方法」同时出现、**`~MyInt` 近似类型**、多接口组合等，规则更细，见 [06](./06-泛型类型约束进阶.md)。

---

## 6. 对照表（背诵用）

| 约束 | 典型用途 |
|------|----------|
| `any` | 通用容器、打印、透传 |
| `comparable` | 去重、查找、**map 键** |
| `cmp.Ordered` | 比大小、排序（1.21+） |
| 自定义 `interface{ A \| B }` | 只允许多个具体类型 |
| 方法接口 | 必须可调某方法 |

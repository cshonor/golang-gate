# 05 - 泛型类型约束基础

**约束**用来限制类型参数 `T`「允许是哪些类型、能做什么操作」。没有约束时，`T` 往往只能做很有限的事（如传递、打印）；加上约束才能比较、运算、调方法等。**`~` 底层类型、联合约束嵌套、`Integer|Float` 式复用、复杂组合约束**见 [06 - 泛型类型约束进阶](./06-泛型类型约束进阶.md)。

---

## 新手先看：大白话五步（先记人话，再记术语）

### 第一步：先记死一句话

**语言里「出厂就写好名字、大家都能直接用」的，只有两个：`any` 和 `comparable`。**  
就两个，没有第三个「语言自带的第三个名字」。

> 规范书里叫 **预声明（predeclared）**；口语说「自带」「天生」都行，**别和「关键字」死磕**，考试/面试再把词说准即可。

### 第二步：人话翻译这两个自带的

1. **`any`**：**啥类型都行**，不挑。`int`、字符串、结构体……先装进去再说。  
2. **`comparable`**：**能做 `==` / `!=`**，**能当 `map` 的 key** 的那一类。去重函数里要 `map[T]...`，所以 `T` 得是 `comparable`。

这两样：**不是你自己在项目里 `type` 造出来的**，是 Go 语言一开始就有的。

### 第三步：搞懂 `cmp.Ordered`

它**不是**上面那种「语言自带名字」。

它是 **Go 团队在标准库里写好的一页接口**：里面用 **`int | float64 | string | …`** 拼出「能写 `>` `<` 比大小」的类型集合。你要做的是 **`import "cmp"`**，然后写 **`[T cmp.Ordered]`**。

所以脑内可以分三档：

- **`any`、`comparable`**：语言**自带名字**（预声明）。  
- **`cmp.Ordered`**：**标准库帮你写好**的约束，**导入即用**，但**不算语言自带第三个**。  
- **你自己写的 `type X interface { ... }`**：**你项目里的自定义**（官方在别的包里写的也算「库定义」，见下一步）。

### 第四步：三类怎么分（最简单版）

| 类 | 包含什么 | 人话 |
|----|----------|------|
| **① 语言自带** | `any`、`comparable` | 出厂只有两个名字 |
| **② 标准库里现成的** | 如 `cmp.Ordered`，以及老项目里 `x/exp/constraints` 那一套 | **官方已写好**，你负责 `import`，**不用自己拼那一长串联合类型** |
| **③ 你自己写的** | `type Num interface{ int \| float64 }` 等 | **在你代码里**声明的约束接口 |

### 第五步：回答「是不是关键字就三个，剩下自己定义？」

**不对。**

- **只有两个**是语言**预声明、自带名字**的：**`any`、`comparable`**。  
- **`cmp.Ordered` 不是第三种「语言关键字」**，只是 **`cmp` 包里的普通类型名**（背后还是接口 + 类型列表）。  
- **「自定义」**在口语里常指 **你自己写的 `type ... interface`**；标准库里的 **`Ordered`** 更准确叫 **「标准库定义的约束」**。

### 新手口诀（超好记）

**自带只有俩：`any` 随便进，`comparable` 能相等；**  
**`Ordered` 是标准库里拼好的，不算自带第三个。**

## 0. 一句话 + 命名厘清（不是「只有两三种约束」）

**结论**：日常口头说的 **`any` / `comparable` / `cmp.Ordered`** 只是**最常见、最省事**的几种写法；**语法层面**的约束是「**类型集接口**」这一套，可由 **联合 `|`、`~`、方法集、多接口嵌入组合** 自由拼出**无数种**合法约束（能过编译器类型检查为准）。

### 0.1 四个常被混用的名字（先对齐术语）

| 名字 | 它是什么 |
|------|-----------|
| **`any`** | **预声明标识符**，`interface{}` 的别名：「任意类型」。 |
| **`comparable`** | **预声明接口约束**：`==` / `!=` 合法的类型集合（仍受语言规则限制，如切片、map、函数不可作 `comparable` 类型实参）。 |
| **`cmp.Ordered`** | **标准库 `cmp` 里的约束接口**（Go 1.21+）：用 **类型联合** 写出「可 `<` `>` 比较」的类型集；**不是**语言关键字。 |
| **自定义约束** | 任意 `type C interface { ... }`，**远不止「竖线联合」一种**：还可 **要方法**、**写 `~T`**、**多个接口嵌入形成交集**等。 |

> 术语精度：规范里 **`any` / `comparable` 属于 *predeclared*（预声明）***；是否叫「关键字」容易口语化过度，笔记里用 **预声明** 更贴规范。

### 0.2 合法约束怎么归类（便于脑内建索引）

1. **预声明约束用语**：`any`、`comparable`（外加类型参数列表里对 `comparable` 的特殊规则等）。  
2. **标准库现成约束**：`cmp.Ordered`；老项目 **`golang.org/x/exp/constraints`** 里仍有 `Signed`、`Unsigned` 等（新代码优先 **`cmp` / `constraints` 文档** 与当前 Go 版本）。  
3. **自建约束接口（类型集）**，常见子类：  
   - **联合**：`int \| int64 \| string`  
   - **近似**：`~int`、`~string`（允许自定义命名类型的底层类型）  
   - **方法**：`interface{ String() string }`  
   - **交集**：多个接口类型嵌入同一约束（`interface{ A; B }` 表示方法集/类型集取交，能否实例化由编译器判定）  

更细的 **core type、联合里塞接口、可实例化** 见 [06](./06-泛型类型约束进阶.md)。

### 0.3 三个常见误区

1. **`cmp.Ordered` 不是「内置第三种关键字」**——它是 **`cmp` 包里的接口定义**，源码里就是 **`Ordered` 接口 + 类型列表**；只有 **`any` / `comparable`** 属于语言**预声明**层面的特殊名字。  
2. **自定义约束 ≠ 只会写 `|`**——方法约束、`~`、交集组合都是日常工程会遇到的。  
3. **90% 业务**确实 **`any` / `comparable` / `cmp.Ordered` 够用**；但面试/读标准库源码时，要上升到 **「类型集接口」** 这一层，避免把 Go 约束想窄了。

### 0.4 速记代码（对照）

```go
// 预声明 comparable：去重、map[K]V 的 K、Contains 等
// func Deduplicate[T comparable](s []T) []T

// 标准库 cmp.Ordered：可三向比较（Go 1.21+）
import "cmp"

func Max[T cmp.Ordered](a, b T) T {
	if a > b {
		return a
	}
	return b
}

// 自定义：仅类型联合
type Num interface{ int | float64 }

// 自定义：方法约束（与下文「方法约束」一节同一思想；命名勿与标准库 fmt.Stringer 混淆亦可）
type HasString interface{ String() string }

// 自定义：底层为 int 的一族（含 type MyInt int）
type IntLike interface{ ~int }
```

## 1. 内置常用约束

### `any`

任意类型，等价于 `interface{}`。

```go
func Print[T any](v T) {
	fmt.Println(v)
}
```

### `comparable`

可用 `==` / `!=` 的类型；**map 的键**必须满足 `comparable`。

```go
func Contains[T comparable](list []T, target T) bool {
	for _, v := range list {
		if v == target {
			return true
		}
	}
	return false
}
```

---

## 2. 标准库 `cmp.Ordered`（Go 1.21+）

需要**比较大小**（如 `>`）时，优先用 **`cmp.Ordered`**（**不要**误用 `comparable` 来写 `>`）。

```go
import "cmp"

func Max[T cmp.Ordered](a, b T) T {
	if a > b {
		return a
	}
	return b
}
```

> **Go 1.18～1.20** 可继续使用 `golang.org/x/exp/constraints` 中的 `constraints.Ordered`，或手写 `int | int64 | string` 等联合类型。

---

## 3. 联合类型约束（`|`）写法详解

Go 1.18+ 里，**类型联合 `T1 | T2` 只能写在「用作类型参数约束的接口」里**（规范里常叫**约束接口** / 带**类型集**的接口）。这是语法硬规则：**不能**给普通变量声明成 `int | string`，**不能**把函数参数写成 `func f(x int | string)`；只有 **`type C interface { ... }` 且该接口出现在泛型约束位置**时，接口体里才可以出现 `|` 类型列表。

### 3.1 基础写法：类型联合

```go
// 约束接口：竖线联合只能出现在这里（及同类 type 定义中）
type Num interface {
	int | int8 | int32 | int64 | float32 | float64
}

func Sum[T Num](nums []T) T {
	var sum T
	for _, v := range nums {
		sum += v
	}
	return sum
}
```

**要点**：`Num` 描述的是「`T` 的类型集」——`T` 只能是列表里**某一种**具体类型；函数体内对 `T` 做的运算（如 `+=`），必须对**联合里每一种类型都合法**。

### 3.2 非法：在非约束场景写联合类型

```go
// ❌ 形参不能写成联合类型
// func f(x int | string) {}

// ❌ 变量也不能声明成联合类型
// var y int | string = 10
```

**原因（人话）**：Go 里每个值在编译期都有**单一确定类型**；`|` 联合只用于**约束类型参数**「允许实例化成哪些类型」，不是一种新的「运行时联合类型」。

### 3.3 坑：联合成员要能支撑函数体里的操作

联合里即使**语法合法**，若成员之间**没有共同可用的运算**，函数体仍会报错：

```go
// 接口本身能通过编译，但下面 Max 里对 T 用 > 会出问题
type Bad interface {
	int | []int
}

// func Max[T Bad](a, b T) T {
// 	if a > b { // []int 不可比较：编译失败
// 		return a
// 	}
// 	return b
// }
```

**思路**：要么缩小联合（例如只用 `cmp.Ordered` 涵盖的可比大小类型），要么改写 API（不要假设所有 `T` 都能 `>`）。

### 3.4 与 `~`、混合约束、嵌套复用

- **`~int`**（近似类型）：把「底层是 `int` 的自定义类型」也收进类型集——见 [06 - 泛型类型约束进阶](./06-泛型类型约束进阶.md) **§一**。  
- **同一接口里「类型列表 + 方法」**：同时要求「属于某几种类型之一」且「实现某方法」——见 [06](./06-泛型类型约束进阶.md) **§三**。  
- **`Integer | Float` 式嵌套**：把常用联合拆成小接口再组合——见 [06](./06-泛型类型约束进阶.md) **§二**。

### 3.5 极简对照（背诵用）

| 写法 | 示例 | 典型用途 |
|------|------|----------|
| 基础联合 | `type N interface{ int \| float64 }` | 限定 `T` 只能是几种**具体**类型 |
| 近似 | `type L interface{ ~int }` | 连 `type MyInt int` 一起收进来（规则见 06） |
| 混合 | `type C interface{ int \| string; String() string }` | 「类型集 ∩ 方法集」更严（慎用） |
| 嵌套 | `type N interface{ Integer \| Float }` | 复用已有约束接口 |

### 3.6 书写注意（与标准库配合）

1. **要比 `<` `>`**：Go 1.21+ 优先 **`cmp.Ordered`**，少手写一长串 `int | int64 | string | …`。  
2. **要 `==` / 或 map 的键**：用预声明 **`comparable`**。  
3. **常用联合**可提成独立 `type`，避免到处复制粘贴。  
4. **联合里每一种**在业务上若都重要，**单测**尽量覆盖到代表类型。

## 4. 方法约束

要求 `T` 必须实现某接口：

```go
type Stringer interface {
	String() string
}

func PrintStr[T Stringer](v T) {
	fmt.Println(v.String())
}
```

---

## 5. 组合约束与 `~`（提要）

「类型列表 + 方法」同时出现、**`~int` 近似类型**、**`Integer | Float` 嵌套**、多接口组合等，规则更细，见 [06](./06-泛型类型约束进阶.md)。

> **`~` 不能乱套**：`~` 只能用在**允许的类型项**上；例如对**命名结构体类型**写 `~MyStruct` 往往会得到 **`invalid use of ~`**（底层类型不是合法的那类时）。细节以编译器报错与规范为准，见 [06](./06-泛型类型约束进阶.md)。

## 6. 对照表（背诵用）

| 约束 | 典型用途 |
|------|----------|
| `any` | 通用容器、打印、透传 |
| `comparable` | 去重、查找、**map 键** |
| `cmp.Ordered` | 比大小、排序（1.21+） |
| 自定义 `interface{ A \| B }` | 只允许多个具体类型（`|` 仅在此类约束接口中） |
| 方法接口 | 必须可调某方法 |

# 07 - 泛型方法（新手向）

> **05-Go Generics**  
> 前置：[04-泛型结构体](./04-泛型结构体.md)、[06-泛型类型约束进阶](./06-泛型类型约束进阶.md) · 下一篇：[08-泛型与标准库结合](./08-泛型与标准库结合.md)

---

## 零、先记一句最核心的（人话）

**泛型方法 = 写在「带 `[T]` 的类型」上的方法**——入门阶段你遇到的几乎都是 **`Box[T]`、`Stack[T]` 这种泛型结构体**，所以你可以先记成：

**给泛型结构体配的函数，就是泛型方法。**

> 严格说：**其它带类型参数的类型**也可以有方法；先把 **`struct` + `[T]`** 吃透就够用了。

---

## 一、长什么样？

**先有**带类型参数的类型，**再写**方法；方法接收者后面也要带 **同样的 `[T]`（或 `[K, V]`）**，和类型定义**对齐**。

```go
type Box[T any] struct {
	value T
}

func (b *Box[T]) Set(v T) {
	b.value = v
}
```

你只要盯死一件事：**`Box` 后面有方括号 `[T]`，方法接收者 `Box` 后面也要跟 `[T]`**，不能丢。

---

## 二、为啥常见 `*Box[T]`？（改值要指针）

**人话**：**要改结构体里面的字段**，接收者一般用 **指针** `*Box[T]`，否则可能改的是**拷贝**，外面看不见变化。

- **`Set` 改 `value`** → 用 **`(b *Box[T])`**  
- **`Get` 只读返回** → 用 **`(b Box[T])`** 也可以（看习惯；大结构体仍常选指针）

---

## 三、最小完整例子（可直接跑）

```go
package main

import "fmt"

type Box[T any] struct {
	value T
}

func (b *Box[T]) Set(v T) {
	b.value = v
}

func (b Box[T]) Get() T {
	return b.value
}

func main() {
	box := Box[int]{}
	box.Set(100)
	fmt.Println(box.Get()) // 100
}
```

**用的时候**：`Box[int]` 把 **`T` 定成 `int`**，方法里 **`T` 就全是 `int`**。

---

## 四、多类型参数（Key-Value）

```go
type KV[K comparable, V any] struct {
	key   K
	value V
}

func (kv KV[K, V]) Print() {
	fmt.Printf("key:%v, value:%v\n", kv.key, kv.value)
}
```

**人话**：结构体是 **`KV[K, V]`**，方法接收者也写 **`KV[K, V]`**（或 `*KV[K, V]`），**`K`、`V` 要和定义一模一样**。

---

## 五、常用：泛型栈上的方法

```go
type Stack[T any] struct {
	items []T
}

func (s *Stack[T]) Push(item T) {
	s.items = append(s.items, item)
}

func (s *Stack[T]) Pop() (T, bool) {
	if len(s.items) == 0 {
		var zero T
		return zero, false
	}
	i := len(s.items) - 1
	v := s.items[i]
	s.items = s.items[:i]
	return v, true
}
```

**人话**：**`Stack[int]`、`Stack[string]`** 都是同一套 `Push`/`Pop`，换 **`T`** 就换栈里元素类型。

---

## 六、新手必背 3 句口诀

1. **泛型方法必须挂在「带 `[T]` 的类型」上**（入门先记：**泛型结构体**）。  
2. **接收者上的 `[T]`（或 `[K,V]`）要和类型定义对齐**，不能写丢。  
3. **要改里面字段 → 多用 `*Foo[T]` 指针接收者**。

---

## 七、两句话分清：泛型函数 vs 泛型方法

| | 泛型函数 | 泛型方法 |
|--|----------|----------|
| **长在哪** | **`func` 名字后面**直接 `[T any]` | 挂在 **`type Foo[T]`** 上，接收者写 **`Foo[T]` / `*Foo[T]`** |
| **怎么调用** | **`Max(1, 2)`**、`**Print(10)**` | **`box.Set(100)`**，前面先有变量 |

**人话**：一个**自己单独能调用**；一个**必须跟着某个「盒子」变量走**。

---

## 八、核心规则（进阶一点，第二遍读）

1. 接收者必须是**带类型参数**的类型。  
2. 方法里用的 **`T`/`K`/`V`**，来自**类型定义**，和接收者一致。  
3. **不能**在「普通不带类型参数的类型」上，再给方法单独发明一套新的 **`[T]`**（和泛型函数不是一回事）。

---

## 附：泛型一条龙五句（02～08 串一行）

1. **`[T …]`** 是类型空位：在**函数名后**是泛型函数（[02](./02-泛型语法基础.md)、[03](./03-泛型函数.md)），在 **`type` 名后**是泛型类型（[04](./04-泛型结构体.md)）。  
2. **`any` / `comparable` / `cmp.Ordered`** 人话版在 [05](./05-泛型类型约束基础.md)。  
3. **`~int`** 让自定义 `type MyInt int` 也能进整数联合（[06](./06-泛型类型约束进阶.md)）。  
4. **方法**：接收者 **`(*Foo[T])`** 上的 **`[T]`** 要和结构体对齐（本篇）。  
5. **日常写业务**：多直接用 **`slices`/`maps`/`cmp`**（[08](./08-泛型与标准库结合.md)）。

| 记什么 | 记一句 |

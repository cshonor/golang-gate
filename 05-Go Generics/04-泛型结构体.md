# 04 - 泛型结构体（新手向）

> **05-Go Generics**  
> 前置：[02-泛型语法基础](./02-泛型语法基础.md)、[03-泛型函数](./03-泛型函数.md) · 约束人话：[05-泛型类型约束基础](./05-泛型类型约束基础.md) · 下一篇：[07-泛型方法](./07-泛型方法.md)

在 **`type` 上**写 `[T 约束]`，让**同一套字段、同一套方法**，既能装 `int`、也能装 `string`……不必为每种类型各造一个结构体。

---

## 一、泛型结构体是干嘛的？（人话）

以前思路常常是：

- 想存 **`int`**，写一个结构体；  
- 想存 **`string`**，再写一个很像的……

**泛型结构体**：**一个结构体模板**，里面的「某种类型」先留个空位叫 **`T`**，以后你说 **`Box[int]`** 还是 **`Box[string]`**，再把它定死。

---

## 二、语法长啥样？

```go
type 名字[T 约束] struct {
	字段 T
}
```

- **`T`**：类型占位符（名字常用 `T`、`K`、`V`，随你习惯）。  
- **`any`**：新手先记这句——**什么具体类型都能当 `T`**。

---

## 三、最简单例子：`Box`

```go
package main

import "fmt"

type Box[T any] struct {
	Content T
}

func main() {
	intBox := Box[int]{Content: 100}    // 装 int：把 T 定成 int
	strBox := Box[string]{Content: "go"} // 装 string：把 T 定成 string
	fmt.Println(intBox, strBox)
}
```

人话：

- **`Box[int]`**：专门装 `int` 的盒子。  
- **`Box[string]`**：专门装 `string` 的盒子。  
- **同一套 `Box` 定义**，换方括号里的类型就行。

---

## 四、泛型结构体 + 方法

```go
type Container[T any] struct {
	data T
}

func (c *Container[T]) Set(v T) {
	c.data = v
}

func (c Container[T]) Get() T {
	return c.data
}
```

人话：

- **`Container` 是什么 `T`**，`Set` 里就**只能传同一个 `T`**，类型不会乱。  
- 接收者写成 **`*Container[T]`** / **`Container[T]`**，意思是「我还在用结构体上的这个 `T`」。

更细的规则见 [07 - 泛型方法](./07-泛型方法.md)。

---

## 五、多类型参数（Key + Value）

```go
type KeyValue[K comparable, V any] struct {
	Key   K
	Value V
}
```

人话：

- **`K`**：当「键」用，要能 **`==` 比较**，所以是 **`comparable`**（和 `map` 的键同一套要求）。  
- **`V`**：值，先用 **`any`**：啥类型都行。

`comparable` 人话版见 [05 篇首](./05-泛型类型约束基础.md)。

---

## 六、常用：泛型栈

```go
type Stack[T any] struct {
	items []T
}

func (s *Stack[T]) Push(v T) {
	s.items = append(s.items, v)
}

func (s *Stack[T]) Pop() (T, bool) {
	if len(s.items) == 0 {
		var zero T
		return zero, false
	}
	i := len(s.items) - 1
	v := s.items[i]
	s.items = s.items[:i]
	return v, true
}
```

人话：

- **`Stack[int]`**：`int` 栈；**`Stack[string]`**：`string` 栈。  
- **一套代码**，换 `T` 就换「栈里装的元素类型」。

---

## 七、两句话分清：泛型函数 vs 泛型结构体

1. **泛型函数**：在**函数名后面**加 `[T any]`，重点是「**这一段逻辑**」对多种类型复用。  
2. **泛型结构体**：在 **type 名字后面**加 `[T any]`，重点是「**这一块数据**」里字段的类型可以随 `T` 变。

一个管「**干啥**」，一个管「**存啥**」；经常搭配用，但**方括号出现的位置不一样**。

---

## 超级简单总结（背这个就够）

1. **泛型结构体** = **一个结构模板**，多种具体类型都能套。  
2. **写法**：`type 名字[T any] struct { ... }`。  
3. **用法**：`名字[int]{}`、`名字[string]{}`——**在类型名后面用方括号把 `T` 定死**。  
4. **方法**：接收者带上同一个 **`[T]`**，例如 `(c *Container[T])`。

---

## 复习速记（表格版）

| 记什么 | 记一句 |
|--------|--------|
| 声明 | `type Foo[T any] struct { ... }` |
| 实例化 | `Foo[int]{...}`、`Foo[string]{...}` |
| 方法 | `(c *Foo[T]) Method(...)` |

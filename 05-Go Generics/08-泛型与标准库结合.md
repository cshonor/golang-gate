# 08 - 泛型与标准库结合

**Go 1.21+** 起，`slices`、`maps`、`cmp` 等包提供大量**泛型 API**，是日常写业务时接触泛型最多的场景。

前置：[02 - 泛型语法基础](./02-泛型语法基础.md)、[03 - 泛型函数](./03-泛型函数.md)、[05 - 泛型类型约束基础](./05-泛型类型约束基础.md)。

---

## 新手先看：一句话总纲（先背这个）

官方帮你备好了 **3 个超好用的泛型工具包**（都是**写一次、多类型复用**）：

1. **`slices`**：给**切片**用——排序、查找、克隆、删段、反转……  
2. **`maps`**：给 **map** 用——克隆、相等判断等。  
3. **`cmp`**：给**三向比较、比大小**用——并附带约束类型 **`cmp.Ordered`**（见 [05](./05-泛型类型约束基础.md)）。

它们**内部全是泛型函数** → 你不用为 `[]int`、`[]string` 各写一套排序。

---

## 1. `slices`（最常用）

**人话**：切片的「万能工具箱」。

```go
import "slices"

a := []int{3, 1, 2}
slices.Sort(a)             // 原地排序（元素需满足包对「可排序」的要求）
_ = slices.Contains(a, 2)   // 是否包含（元素要 comparable）
_ = slices.Clone(a)       // 浅拷贝新切片（改副本不影响 a）

// Delete 会删掉 [i,j)，**返回值要赋回原变量**
a = slices.Delete(a, 0, 1)

xs := []int{1, 2, 3}
slices.Reverse(xs) // 原地反转 → 3,2,1

// 二分前切片需已有序
ys := []int{10, 20, 30}
_, _ = slices.BinarySearch(ys, 20)
```

### 为什么是泛型？

同一套 API 能服务 **`[]int`、`[]string`、`[]float64`**……底层用类型参数把「元素类型 `E`」抽象掉；**排序类**往往要求 **`cmp.Ordered`**，**Contains / Compact** 等常要求 **`comparable`**——约束在函数签名里都写好了，见 [05](./05-泛型类型约束基础.md)。

---

## 2. `maps`

**人话**：map 的常用操作，少手写循环。

```go
import "maps"

m := map[string]int{"a": 1}
m2 := maps.Clone(m) // 浅拷贝

// m1、m2 键值类型需满足 Equal 的约束（键、值都可比较）
// var m1, m2 map[string]int
// _ = maps.Equal(m1, m2)
```

### 为什么是泛型？

**键**在 Go 里本身就必须 **`comparable`**；`maps` 再把「**键类型 K、值类型 V**」抽成类型参数，所以 **`map[string]int`、`map[int]bool`** 都能用同一套函数。

---

## 3. `cmp`

**人话**：专门做**比较**；和排序、二分、`Min`/`Max` 模板是一伙的。

```go
import "cmp"

_ = cmp.Compare(1, 2) // -1 / 0 / 1
_ = cmp.Less(3, 5)    // true
```

**`cmp.Ordered`**：标准库里定义好的**约束接口**，表示「能用 `<` `>` 排一队」的类型集（`int`、`float64`、`string` 等在列）。你写泛型时写 **`[T cmp.Ordered]`**，就和 `slices.Sort`、`slices.BinarySearch` 等**同一套规矩**，见 [05](./05-泛型类型约束基础.md)。

---

## 4. 和「泛型」到底是什么关系？（超短人话）

**泛型** = **一套实现，多类型复用**。

以前容易想到「**每种类型抄一遍**」：

```go
// 伪代码：示意「没有泛型时重复多」
// sortInts([]int)
// sortStrings([]string)
```

现在标准库用泛型：**你传 `[]T`，编译器按 `T` 实例化**；业务里多数时候**不用写 `[T]`**，类型推断就够了（见 [02](./02-泛型语法基础.md)、[03](./03-泛型函数.md)）。

```go
xs := []int{3, 1, 2}
slices.Sort(xs) // 推断 T 为 int；底层等价于「对 Ordered 元素排序」那一类 API
```

---

## 5. `errors` 与多错误（提要）

`errors.Join`（Go 1.20+）与泛型**无直接语法关系**，但在 **errgroup / 并发聚合** 场景常与通用工具一起出现；见 [03-error_handling](../03-error_handling/README.md)。

---

## 6. 使用建议

- **优先调标准库**，少自造轮子；读签名等于复习 **`comparable` / `cmp.Ordered`**。  
- 需要兼容旧版本时，对照 **`golang.org/x/exp/slices`** 等迁移路径与发行说明。

---

## 7. 一页速记（面试 + 日常）

| 包 | 记住一句 | 典型 API / 概念 |
|----|----------|-----------------|
| **`slices`** | 切片工具箱 | `Sort` `Contains` `Clone` `Delete` `Reverse` `BinarySearch` … |
| **`maps`** | map 克隆 / 比较 | `Clone` `Equal`（注意 **值也要 comparable** 才能 `Equal`） |
| **`cmp`** | 比大小 + 约束名 | `Compare` `Less`、约束 **`cmp.Ordered`** |

**背两句就够**：

- **泛型**：写一次逻辑，**多类型**用。  
- **`slices` / `maps` / `cmp`**：官方替你写好的**泛型标准库**。

# 03 - 泛型函数（新手向）

> **05-Go Generics**  
> 前置：[02-泛型语法基础](./02-泛型语法基础.md) · 下一篇：[04-泛型结构体](./04-泛型结构体.md) · 约束人话版：[05-泛型类型约束基础](./05-泛型类型约束基础.md)

你已经知道：**方括号 `[T any]` 是给类型留空位**。本篇只讲一件事：**这种空位写在「函数」上时，长什么样、怎么调用**。

---

## 零、先记一句话（泛型函数是啥？）

**一个函数，可以接住 `int`、`string`、`float64` 等各种类型，而不用为每种类型各写一版几乎一样的代码。**

---

## 一、和普通函数差在哪？

普通函数：

```go
func SayHi(name string) { ... }
```

泛型函数：**在函数名后面多一对方括号**，里面写上「类型空位 + 约束」：

```go
func SayHi[T any](name T) { ... }  // 仅作形状示例，一般用 string 即可
```

**调用时**：多数情况**不用**写 `[string]`，编译器会从实参**推断** `T` 是什么。

---

## 二、基本形状（先认脸）

```go
func 函数名[类型参数 约束](参数列表) 返回值 {
	// 函数体
}
```

人话对照：

- **`T`**：就是一个**类型占位符**，意思是「这里有一种类型，先起名叫 `T`」。  
- **`约束`**：限制「`T` 允许是哪些类型、能做什么运算」（最常见先记 **`any`** 和 **`cmp.Ordered`**）。

例子（最常用的形态）：

```go
func Print[T any](v T) {
	// ...
}
```

- **`Print`**：函数名。  
- **`[T any]`**：类型参数列表；这里只有一个 `T`，约束是 `any`。  
- **`(v T)`**：普通参数；类型用的是 **`T`**。

---

## 三、`any`：任意类型（和 02 同一例子）

```go
package main

import "fmt"

func Print[T any](v T) {
	fmt.Println(v)
}

func main() {
	Print(10)       // T 推断为 int
	Print("hello")  // T 推断为 string
	Print(3.14)     // T 推断为 float64
}
```

**要点**：三行调用**没有**写 `Print[int](10)` 也能过——这叫**类型推断**。

---

## 四、带约束的函数：比大小用 `cmp.Ordered`（Go 1.21+）

要写 **`a > b`**，就不能只用 `any`（因为不是所有类型都能比大小）。常见写法：

```go
import "cmp"

func Max[T cmp.Ordered](a, b T) T {
	if a > b {
		return a
	}
	return b
}
```

- **`cmp.Ordered`** 不是「语言自带第三个名字」，是**标准库里拼好的约束**；人话版见 [05 篇首](./05-泛型类型约束基础.md)。  
- **Go 1.18～1.20**：可用 `golang.org/x/exp/constraints` 里的 `Ordered`，或自己写 `int | int64 | ...` 联合类型。

---

## 五、自定义约束：只允许某几种类型（联合）

```go
type Number interface {
	int | int64 | float64
}

func Add[T Number](a, b T) T {
	return a + b
}
```

意思是：**`T` 只能是 `int`、`int64`、`float64` 里的一种**；`Add(1, 2)` 可以，`Add(1, int64(2))` 不行（两个参数必须是**同一个** `T`）。

---

## 六、多个类型参数

空位可以不止一个，用逗号分隔：

```go
func Pair[A, B any](a A, b B) (A, B) {
	return a, b
}
```

- **`A`、`B`**：两个类型空位，这里都约束成 `any`。  
- 调用 `Pair(1, "x")` 时，通常推断为 **`A=int`，`B=string`**。

---

## 七、什么时候要自己写 `[具体类型]`？（知道有这回事）

多数情况**不用写**。下面两种场景你会见到显式写法：

1. **编译器推断不出来**（相对少见，先有个印象即可）。  
2. **你想写清楚**（可读性 / 与复杂表达式搭配时）：`Print[int](10)`。

第一遍**不用背**；报错里出现 `cannot infer T` 再查即可。

---

## 八、三句话分清 `any`、`comparable`、`cmp.Ordered`（背了不混）

1. **`any`**：**啥类型都能当 `T`**，不挑；只管传、只管存，不要求 `>` 或当 `map` 键。  
2. **`comparable`**：**能做 `==` / `!=`**，**能当 `map` 的 key**；要写「去重、计数、`map[T]…`」常会碰到。  
3. **`cmp.Ordered`**：**标准库里已经拼好的一页接口**，表示「允许用 `>` `<` 比大小」的那几种类型；写 `Max`、`Sort` 这类才需要。

更细的人话分层见 [05 篇首「大白话五步」](./05-泛型类型约束基础.md)。

---

## 超级简单总结（背这 6 句就够）

1. **泛型函数** = 一个函数适配多种类型。  
2. **`[T 约束]`** 写在**函数名后面**。  
3. **`any`** = 啥类型都行。  
4. **`cmp.Ordered`** = 能比大小（`>` `<` 等，Go 1.21+ 用标准库 `cmp`）。  
5. **自定义约束** = 自己写 `type X interface { int | float64 | ... }`，规定 `T` 只能是哪几种。  
6. **`A, B any`** = 同时留两个类型空位（例如一个 `int` 一个 `string`）。

---

## 复习速记（表格版）

| 记什么 | 记一句 |
|--------|--------|
| 写在哪 | 类型参数在**函数名后面**：`func F[T any](...)` |
| 调用 | 多数只写 `F(实参)`，让编译器推断 `T` |
| 比大小 | Go 1.21+ 用 **`[T cmp.Ordered]`** |

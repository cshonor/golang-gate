# 01 - 泛型介绍（为什么需要）

**泛型**让你写**一份**代码，对多种具体类型复用，同时保持**编译期类型检查**，一般也比 `interface{}` + 类型断言 / 反射更易读、更快。

## Go 1.18 之前常见做法

- `interface{}`（`any`）：容易丢类型信息，要靠断言，**运行时**才发现错。
- **反射**：灵活但慢、代码啰嗦、易 panic。
- **复制粘贴**：为 `int`、`int64` 各写一版 `Max`，维护成本高。

## Go 1.18+ 泛型带来的变化

- **类型安全**：错误尽量在**编译**阶段拦住。
- **性能**：编译器为不同类型参数生成代码（单态化思路），通常接近手写具体类型。
- **复用**：一个 `Max[T]`、`List[T]` 多类型共用。

## 核心概念（先记三个词）

1. **类型参数**：如 `func F[T any](x T)` 里的 `T`。
2. **约束**：限制 `T` 能是什么（`any`、`comparable`、自定义 `interface{ ... }` 等）。
3. **泛型函数 / 泛型类型**：函数或 `type Box[T any] struct { ... }` 这类带类型参数的定义。

## 最小例子

```go
import "fmt"

func Print[T any](v T) {
	fmt.Println(v)
}

func main() {
	Print(10)
	Print("hello")
}
```

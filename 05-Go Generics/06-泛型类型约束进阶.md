# 06 - 泛型类型约束进阶（新手向 + 提要）

> **05-Go Generics**  
> 前置：[05-泛型类型约束基础](./05-泛型类型约束基础.md)（含篇首「大白话五步」）、[02-泛型语法基础](./02-泛型语法基础.md)、[04-泛型结构体](./04-泛型结构体.md) · 下一篇：[07-泛型方法](./07-泛型方法.md)

本篇只多记几件事：**波浪号 `~`、少用的「类型+方法」组合、以及和 `comparable` 相关的直觉**。第一遍**只读「零～三」+ 超级总结**也完全够。

---

## 零、新手先只背这两句

1. **`~int`**：**所有「底层本质是 `int`」的类型**都算——包括你自己写的 `type MyInt int`。  
2. **联合里只写 `int`、不写 `~`**：**基本只有 `int` 自己**；你的 `MyInt` **进不来**。

---

## 一、`~` 波浪号是什么？（人话）

### 大白话

**`~int` = 所有「骨子里是 `int`」的类型**，包括：

- 原生 **`int`**  
- 你自己写的 **`type MyInt int`**  
- 其它**底层类型（underlying type）是 `int`** 的有名类型

### 和只写 `int` 的差别

| 写法 | 人话 |
|------|------|
| **`int`（在联合里）** | 基本就是 **`int` 本家** |
| **`~int`** | **`int` 本家 + 所有底层是 `int` 的自定义类型** |

### 一看就懂的例子

```go
type MyInt int

// 只有 int：MyInt 通常不满足「只是 int」那种联合（教学对比用）
type OnlyInt interface{ int }

// 带 ~：MyInt 也能算「整数这一类」
type Intish interface {
	~int | ~int8 | ~int16 | ~int32 | ~int64
}

func Double[T Intish](x T) T {
	return x * 2
}

var _ = Double(MyInt(3)) // OK：MyInt 底层是 int
```

**一句话**：想让 **`type MyInt int`** 这种也算进去，联合里就写 **`~int`**（同理 `~string` 等）。

---

## 二、「类型列表 + 方法」组合约束（知道有这回事就行）

**人话**：既要「属于某几种类型之一」，又要「带某个方法」——约束变严，**新手很容易写出编译器说不行的组合**。

```go
type MyConstraint interface {
	int | string
	String() string
}
```

**新手策略**：这种写法**先别急着用**；日常更稳的是 **要么只写类型联合**，**要么只写纯方法接口**（见 [05](./05-泛型类型约束基础.md)）。真要混用，等报错 **`T does not satisfy`** 再对照编译器提示改。

---

## 三、`comparable` 到底是啥？（你多半已经见过）

**人话**：**能用 `==` / `!=` 比「两个值一不一样」的类型**（语言有明确规则，不是「我觉得能比就能比」）。

先记这些直觉（够应付大部分题）：

| 常见类型 | 能不能当「可比」的典型直觉 |
|----------|----------------------------|
| `int`、`string`、指针等 | 通常 ✅ |
| **切片 `[]T`** | ❌（不能当 `comparable` 类型实参） |
| **map** | ❌ |
| **函数** | ❌ |
| **结构体** | **要看字段**：**全都能比**的 struct 才可以；记不住就记 **`map` 的键必须是 `comparable`** |

**用途**：**`map[K]V` 的 `K`** 必须是 `comparable`。

更多「出厂自带 vs `cmp.Ordered`」见 [05 篇首](./05-泛型类型约束基础.md)。

---

## 四、类型集（第二遍再记这个词）

约束接口其实是在描述一个**类型集合**：`T` 必须是里面的成员。**`~`** 的作用，就是把这个集合从「只有 `int`」扩成「**所有底层是 `int` 的**」。

设计文档：[Type Parameters Proposal](https://go.googlesource.com/proposal/+/refs/heads/master/design/43651-type-parameters.md)。

---

## 五、常见编译错误（对照用）

| 现象 | 常见原因 |
|------|----------|
| `T` does not satisfy `X` | 类型不在联合里、或方法集不满足 |
| 自定义 `MyInt` 传不进 `int \| int64` | 改成 **`~int \| ~int64`** 等 |
| 组合约束总报错 | **类型+方法** 太严，先拆成单一风格 |

---

## 超级简单总结（背这个就够）

- **`~类型`**：**连自定义的同底层类型**一起算进去。  
- **`comparable`**：**能 `==`**；**`map` 的键**需要它。  
- **`any`**：**不挑类型**（见 [05](./05-泛型类型约束基础.md)）。  
- **`cmp.Ordered`**：**能比大小**（标准库 `cmp`，见 [03](./03-泛型函数.md)）。

---

## 附：02～07 五句新手口诀（可选，一条线串起来）

1. **`[T …]`** 是**类型空位**：写在**函数名后**是泛型函数，写在 **`type` 名后**是泛型类型（见 [02](./02-泛型语法基础.md)、[03](./03-泛型函数.md)、[04](./04-泛型结构体.md)）。  
2. **`any` / `comparable`**：语言里**最常挂嘴边**的两个约束名字（见 [05](./05-泛型类型约束基础.md)）。  
3. **`cmp.Ordered`**：要比 **`>` `<`**，从标准库 **`cmp`** 拖这个（见 [03](./03-泛型函数.md)）。  
4. **`~int` 这类**：让 **`type MyInt int`** 也能混进「整数一族」联合里（本篇 **§一**）。  
5. **「类型 + 方法」硬组合**：**先别碰**，等真需要再学（本篇 **§二**）。  
6. **方法接收者**：带上同一个 **`[T]`**，别写乱（见 [07](./07-泛型方法.md)）。

（上面是 **6 句** 把 **07** 也收进来；若你只想背 **5 句**，最后一条可以等打开 **07** 再背。）

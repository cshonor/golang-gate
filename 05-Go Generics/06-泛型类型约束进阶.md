# 06 - 泛型类型约束进阶

前置：[05 - 泛型类型约束基础](./05-泛型类型约束基础.md)、[02 - 泛型语法基础](./02-泛型语法基础.md)。

---

## 1. `~`：允许底层类型匹配

`~T` 表示：**以 `T` 为底层类型的自定义类型**也满足约束。

```go
type MyInt int

type Intish interface {
	~int | ~int8 | ~int16 | ~int32 | ~int64
}

func Double[T Intish](x T) T {
	return x * 2
}

var _ = Double(MyInt(3)) // OK：MyInt 底层类型是 int
```

没有 `~` 时，联合里写 `int` **不包含** `MyInt`。

---

## 2. 联合约束里的规则（提要）

- 联合项通常是**具体类型**或 **`~T`**；需满足语言对 **core type**、**方法集** 的检查。
- **接口作为联合成员**时规则更严：常见写法是 `interface{ Method() }` 与类型列表组合时，要保证可实例化的类型确实存在（否则编译失败）。

---

## 3. 组合约束：类型列表 + 方法

```go
type MyConstraint interface {
	int | string
	String() string
}
```

注意：`int`、`string` **预定义类型本身没有** `String()`，因此该约束通常只对那些**自定义有名类型**（底层为 `int`/`string` 且实现了 `String()`）可实例化。工程上更常见、也更稳的是：**要么只用类型列表运算**，**要么只用「纯方法接口」** 做约束；混合时若编译器报 `does not satisfy`，检查联合里是否**每种类型**都能满足方法集。

---

## 4. `comparable` 与 `==`

- `comparable` 包含指针、通道、接口、由可比字段组成的 struct 等；**切片、map、函数类型不可比**（不能作 `comparable` 类型实参）。
- **map 键**：`K` 必须是 `comparable`，与是否 `Ordered` 无关。

---

## 5. 约束中的类型集（类型集语义，面试常提）

Go 泛型里，约束接口定义了一个**类型集合**；`T` 必须属于该集合。`~` 扩展了「与某底层类型一致」的子集。细节以 [Type Parameters Proposal](https://go.googlesource.com/proposal/+/refs/heads/master/design/43651-type-parameters.md) 与当前 `go/types` 行为为准。

---

## 6. 常见编译错误

| 现象 | 常见原因 |
|------|----------|
| `T` does not satisfy `X` | 某具体类型未在联合中、或未实现方法 |
| 自定义类型传不进 `int \| int64` | 需要写成 `~int \| ~int64` 等 |
| 接口约束过宽/过窄 | 类型列表与方法同时约束过强 |

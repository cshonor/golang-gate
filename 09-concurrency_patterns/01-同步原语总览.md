# 01 - 同步原语总览

## 1. 先分清“同步目标”

Go 并发里你做同步，通常就 4 类目标：

1. **互斥**：同一时间只能一个 goroutine 进入临界区（`sync.Mutex` / `sync.RWMutex`）
2. **等待一组任务结束**：`sync.WaitGroup`
3. **只初始化一次**：`sync.Once`
4. **条件等待/事件通知**：`sync.Cond` 或 channel（更常用）

辅助工具（偏工程）：

- **对象复用**：`sync.Pool`
- **超时/取消/链路传递**：`context.Context`

## 2. 什么时候用 channel，什么时候用锁？

经验法则：

- **共享内存 + 临界区**：用锁（mutex/RWMutex）
- **传递数据/编排流程**：用 channel（配合 `select`）

面试表述：

> channel 适合“通信”；锁适合“保护共享状态”。实际工程常组合使用。

## 3. 常见组合（你写服务一定会遇到）

- **context + select**：请求超时/取消、关闭 goroutine
- **WaitGroup + err channel**：并发跑多任务，汇总错误
- **Pool + bytes.Buffer**：减少临时对象分配，降低 GC 压力

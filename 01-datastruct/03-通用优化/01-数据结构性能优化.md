# 01 - 数据结构性能优化（slice / map 通用）

跨 **slice** 与 **map** 的常用优化手段；与 [01-slice/03-slice常见坑与最佳实践](../01-slice/03-slice常见坑与最佳实践.md)、[02-map/08-map内存模型与GC](../02-map/08-map内存模型与GC.md) 呼应。

---

## 1. slice

- **预分配 `cap`**：`make([]T, 0, n)` 或已知长度时 `make([]T, n)`，减少 `append` 扩容与拷贝。  
- **避免无意义子切片常驻**：大 backing 数组问题见 slice 坑篇。  
- **`copy` 替代手动循环**赋值（清晰且常更快）。  
- **大对象复用**：可考虑 `sync.Pool`（仅适合**生命周期短、可重置**的对象，误用会泄漏语义）。

---

## 2. map

- **`make(map[K]V, hint)`**：提示元素量，**减少扩容次数**（hint 非上限）。  
- **key/value 体量**：大 value 评估指针或外置存储。  
- **并发**：不是性能调优万能药——错误并发比「慢」更糟；见 [07-map并发安全方案](../02-map/07-map并发安全方案.md)。

---

## 3. 测量优先

用 **`go test -bench`**、**pprof**、**benchstat** 验证假设；避免仅凭直觉改「性能」。

# 01 - array 数据结构（新手向）

数组是 Go 里最基础的线性结构：**一段连续内存 + 固定长度**。

---

## 1. 数组是什么？（一句话）

**数组类型写成 `[N]T`**：`N` 是长度，**是类型的一部分**。

```go
var a [3]int
var b [4]int
// a = b // 编译不通过：[3]int 和 [4]int 不是同一类型
```

---

## 2. 常见写法（够用版）

```go
var a [3]int           // 全零值
b := [3]int{1, 2, 3}   // 指定所有元素
c := [3]int{1}         // 只给 c[0]，其它为零值
d := [...]int{1, 2, 3} // 让编译器推断长度
```

---

## 3. 赋值/传参：最大区别（面试必问）

**数组是值类型**：赋值、传参时通常会拷贝**全部元素**。

```go
func f(x [3]int) {
	x[0] = 999
}

func main() {
	a := [3]int{1, 2, 3}
	f(a)
	// a 仍然是 [1 2 3]（因为 f 拿到的是副本）
}
```

要在函数里改「外面的数组」，常见两种方式：

1. **传指针**：`func f(x *[3]int)`（会把维度也写死在类型里）。  
2. **用切片**：`func f(x []int)`（更常见，见 `01-slice/`）。

---

## 4. 什么时候用数组？（实用选型）

- **定长语义就是业务的一部分**：如协议头、固定维度矩阵、固定窗口缓存。  
- **需要把“长度”变成类型约束**：让 `[4]byte` 和 `[8]byte` 在类型层面区分开。  
- **小数组**：当作轻量值对象传来传去也 OK；但**大数组传参**要小心拷贝成本（见下一篇）。  

绝大多数动态集合场景：**用切片**。

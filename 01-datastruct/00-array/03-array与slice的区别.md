# Go 数组与切片 (Array vs Slice) 深度对比

## 📌 核心差异
| 特性 | 数组 (Array) | 切片 (Slice) |
| :--- | :--- | :--- |
| **长度** | **固定**，声明后不可变 | **动态**，可灵活扩容 |
| **值/引用** | **值类型**，赋值传参全复制 | **引用类型**，指向底层数组 |
| **容量** | 容量 = 长度 (len = cap) | 容量 >= 长度，可动态增长 |
| **传递效率** | 大数组效率极低 | 传递指针/切片，效率高 |
| **声明方式** | `[n]T` | `[]T` |

## 🔍 底层实现关系
**切片是对数组的抽象与封装**。
- 切片底层依赖一个**底层数组**
- 切片由三部分组成：**指针 (指向底层数组)、长度 (len)、容量 (cap)**
- 当切片扩容时，会创建新的更大数组，并复制数据

## 🎯 适用场景对比
### 1. 数组适用场景
- 长度**固定**且已知的业务数据
- 作为多维数组的维度
- 用于底层内存的极致控制
- 避免切片扩容带来的内存重分配

### 2. 切片适用场景
- 动态长度的数据集
- 需要频繁追加、删除元素
- 函数参数传递（避免大数组复制）
- 循环遍历、数据处理

## 💡 转换与关联
### 1. 数组转切片
```go
arr := [3]int{1, 2, 3}
slice := arr[:] // 转换为切片，指向原数组
```

### 2. 切片转数组（需指定长度）
```go
slice := []int{1, 2, 3}
var arr [3]int
copy(arr[:], slice) // 复制切片数据到数组
```

## ⚠️ 常见误区
1. **误以为切片是数组的引用**：切片是引用类型，但它本身是一个结构体，包含指针、len、cap。
2. **数组传参复制**：直接传大数组会导致性能问题，必须传指针。
3. **切片扩容影响原数组**：切片未扩容时，修改切片元素会影响原数组；扩容后则不会。

## 🔚 总结
数组是基础，切片是工具。
- 数组解决“固定长度、连续内存”的问题
- 切片解决“动态长度、高效传递”的问题
- 99% 的业务开发应优先使用 **切片**，仅在特定底层场景使用数组

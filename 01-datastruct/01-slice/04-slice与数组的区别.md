# 04 - slice 与数组的区别

前置：[01-slice数据结构](./01-slice数据结构.md)。**数组**与**切片**在类型系统与传参语义上不同，面试常考「值拷贝范围」。

---

## 1. 类型本质

| | 数组 `[N]T` | 切片 `[]T` |
|---|-------------|------------|
| 长度 | **编译期固定**，类型的一部分（`[4]int` 与 `[5]int` 是不同类型） | **运行时**由 `len`/`cap` 表示 |
| 赋值 / 传参 | **整体按值拷贝**（拷贝全部元素） | 拷贝的是**切片头**（指针、len、cap），底层数组共享 |

---

## 2. 传参直觉

- **数组**：`func f(a [1024]byte)` 会拷贝 **1024 字节**（大数组成本高）。  
- **切片**：`func f(s []byte)` 只拷**头**，与「引用语义」类似（但 Go 没有引用类型，只是头很小）。

---

## 3. 何时用谁

- **固定长度、栈上小对象、类型即维度**（如矩阵维度、协议定长头）：考虑数组。  
- **动态长度、与标准库 API 对接**：几乎总是 **slice**。

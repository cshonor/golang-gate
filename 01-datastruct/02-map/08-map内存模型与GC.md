# 08 - map 内存模型与 GC（直觉）

前置：[02-map数据结构旧版](./02-map数据结构旧版.md)、[03-map渐进式扩容旧版本](./03-map渐进式扩容旧版本.md)。**细节以 `runtime/map.go` 与当前版本为准**。

---

## 1. 分配与增长

- `map` 按需增长：**桶数组**、**溢出桶**随元素增多而分配；**渐进式扩容**时新旧表会短暂共存（见扩容篇）。  
- 桶与溢出桶由 runtime 管理，**无稳定对外 API** 观察「当前块数」。

---

## 2. 与 GC 的关系

- **key/value 含指针**时，GC 需扫描可达对象；`map` 内部对指针有专门扫描路径（随版本优化）。  
- **大 value**（如很少用的大 `[]byte`）长期留在 map 里 → 与「大对象一直可达」一样占内存。

---

## 3. 「泄漏」直觉

- **未 `delete` 的 key** → 对应 entry 及大 value 一直可达，**不是 runtime bug**，是使用方式问题。  
- 需要「清空」时：遍历 `delete`、或**换一个新的 map** 让旧表整体不可达（由 GC 回收）。

---

## 4. 实践

- 能预估元素量时：`make(map[K]V, hint)` **减少反复扩容**（见 [../03-通用优化/01-数据结构性能优化.md](../03-通用优化/01-数据结构性能优化.md)）。  
- 大对象做 value 时评估是否应放**指针**或外置缓存。

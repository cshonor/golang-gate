# map 并发不安全

用**最直白**的方式记：**原生 `map` 不是并发安全的**；一边写、一边读/写，运行时可能**直接帮你 panic**。底层为啥会炸见下文；**怎么改安全**见 [07-map并发安全方案.md](./07-map并发安全方案.md)。结构背景见 [01-map-hmap与bmap-直觉对照.md](./01-map-hmap与bmap-直觉对照.md)。

---

## 新手先看：终极背诵版（面试 + 工作）

1. **原生 map：并发读写 → 未定义行为**，常见 **`fatal error: concurrent map read/write`**（运行时**专门检测**）。  
2. **原因**：内部要改**桶、指针、count、扩容搬迁**；多 goroutine 同时动 → **竞态**。  
3. **安全路子（粗分）**：普通业务 **`map + sync.RWMutex`**；**读极多写极少**再考虑 **`sync.Map`**；更高吞吐 **`分片 map`**。  
4. **`sync.Map` 不是万能**（API、场景都有限制）。  
5. **`atomic.Value` 可以存 `*map` 做「整体替换」**，**不能**多人并发改**同一张** map 本体。

---

## 1. 核心结论（必须背）

- **多个 goroutine 在未同步情况下对同一 map 又读又写** → **可能直接 fatal**（例如 **`concurrent map read/write`**）。  
- **不要赌「我这会儿好像没崩」**——语言层面是**未定义行为**，线上换负载就炸。

**人话模型（够用版）**：

- **并发只读**（没有任何写入）在**较新 Go 版本**里有更清晰的规则，但**生产里仍别裸依赖**「大家约定都不写」这种默契。  
- **一旦有写**：要么**串行化写**，要么**整段 map 访问都加同步**；**读写在不同 goroutine 交错**最容易踩雷。

---

## 2. 为什么不安全？（人话版）

map 内部会做这些事（和 [01](./01-map-hmap与bmap-直觉对照.md)、[03](./03-map渐进式扩容旧版本.md) 串起来读）：

- **插入 / 删除 / 更新**槽位与 key/value。  
- **渐进式扩容、搬迁桶**，改 **`hmap` 里的指针、`count`、标志位** 等。

如果两个 goroutine **同时改这套结构**：

- 内存里可能看到**半迁移状态**；  
- 指针与计数**对不上**；  
- 运行时检测竞态 → **直接 panic / fatal**。

**一句话**：map 被实现成**「单线程拥有时最快」**的结构，**没给你并发无锁承诺**。

## 4. 两道自检题（面试高频）

### 第一题：`sync.Map` 为什么不是万能替换 `map`？

**答（人话版）**：

1. **没有 `m[k]` 语法糖**，要用 **`Load` / `Store` / `Delete` / `Range`** 一套 API。  
2. **只适合「读多写少、key 集合相对稳定」**一类场景；乱用容易**性能难看**、逻辑也难推理。  
3. **通用读写路径**上，很多时候 **`RWMutex + map`** 更直观、更好测。  
4. **类型层面**：标准库仍是 **`any`** 风格，**没有**「带 `K/V` 类型参数的下标语法那种 `sync.Map`」；别把 **`sync.Map`** 和 **`map[K]V`** 当成同一个东西。

**结论**：**普通业务优先 `map + RWMutex`**；确认是「读极多写极少」再评估 `sync.Map`，并 **benchmark + `-race`**。

---

### 第二题：能不能用 `atomic.Value` 存整个 map 指针，实现「无锁」更新？

**答：可以当一种「发布快照」模式用，但限制很死。**

1. **不能**让多个 goroutine **并发改同一张** map（会回到并发不安全）。  
2. **常见套路**：读方 **`Load`** 指针拿到**只读快照**；写方 **复制/新建一份 map**，改完后 **`Store` 换指针**——本质是 **copy-on-write / 整体替换**。  
3. **适合**：**写极少**、读很多、能容忍**短暂读到旧快照**。  
4. **不适合**：**高频写**（复制整张表太贵）。  

**一句话**：**`atomic.Value` 管的是「指针这一格」的原子切换，不是帮你 magically 并发改 map。**

细节与注意点见 [07-map并发安全方案.md](./07-map并发安全方案.md) 中 **`atomic.Value` 与 map 快照** 一节。

# map 遍历顺序不固定

## 1. 你在代码里看到的现象

- `for k, v := range m` 每次运行 **顺序可能不同**。  
- **不要依赖**遍历顺序写逻辑（例如「先插入的先出来」）。

## 2. 语言层面

- Go **刻意不保证** map 迭代顺序，且历史上通过 **随机起点** 等机制避免开发者依赖顺序（与实现演进有关）。

## 3. 若需要稳定顺序

- 取出 **keys** → `sort.Strings(keys)`（或按业务排序）→ 再按键访问。  
- 需要插入顺序用 **slice + map** 或独立链表结构。

## 4. 与渐进式扩容、乱序原理的关系

- **扩容 / 搬迁** 会改变桶布局；两表共存阶段遍历规则更复杂，顺序更不可依赖 → 见 [03-map渐进式扩容旧版本.md](./03-map渐进式扩容旧版本.md)。  
- 「不固定」是 **语义 + 实现策略**；具体随机机制见 [05-map乱序原理.md](./05-map乱序原理.md)。

# map 乱序原理（为何迭代「看起来随机」）

## 1. 目标

- 防止开发者写出 **依赖遍历顺序** 的程序（顺序无语言保证）。  
- 使依赖顺序的代码更容易在测试/升级中暴露问题。

## 2. 常见实现手段（概念）

- **随机化迭代起点**：每次 `range` 从不同桶/槽位开始扫描。  
- 结合 **随机的 hash seed**（与 key 分布、安全有关），间接影响桶分布与遍历观感。

## 3. 注意区分

- **乱序/随机起点** ≠ map 存取无序：按键查找仍是 O(1) 均摊意义下的哈希表行为。  
- **扩容/搬迁** 也会改变物理布局，使顺序更不可预测。

## 4. 自检

- **若遍历顺序固定**：依赖顺序的错误代码更容易在测试里「一直碰巧通过」，上线后换环境才炸——Go **故意不保证顺序** 正是为了减少这种假设；展开见 [04-map遍历顺序不固定.md](./04-map遍历顺序不固定.md) 文末「自检」一节。  
- **`reflect.DeepEqual` 比较两个 `map`**：**不关心**遍历顺序，只比较**键值对是否一致**（map 相等由键值集合决定，与迭代次序无关）。

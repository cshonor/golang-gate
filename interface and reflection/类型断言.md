# 2. 类型断言（Type Assertion）

「只要实现了接口的方法，就都是这个接口的类型。但是它（接口变量）不知道是哪个结构体实现的这个方法」


// 接口：只定义「会叫」这个契约
type Speaker interface {

    Speak() string

}



// 结构体Dog：实现Speak() → 自动成为Speaker类型

type Dog struct{ 
Name string
 }


func (d Dog) Speak() string {
 return d.Name + " 汪汪"
 }

// 结构体Cat：实现Speak() → 自动成为Speaker类型

type Cat struct{ 
Name string
 }

func (c Cat) Speak() string { 
return c.Name + " 喵喵"
 }


// 把两个结构体都装进接口变量
var s Speaker

s = Dog{Name: "旺财"}  // s现在是Speaker类型，背后是Dog

s = Cat{Name: "咪咪"}  // s现在还是Speaker类型，背后是Cat

接口变量 s 的世界里，只有「Speaker」这个身份，没有「Dog/Cat」的区别：

 
- ✅ 它知道自己能调用 Speak() （因为接口里定义了这个方法）

​
- ❌ 它完全不知道自己背后是Dog还是Cat，更不知道Dog有 Bite() 、Cat有 CatchMouse() 这些专属方法
​

- ❌ 它只认「契约」，不认「真身」，就像你拿到一张「会员卡」，只知道卡的权益，不知道卡是谁办的

## 1）干什么用

把**接口变量**还原成**具体类型**（或**另一接口类型**）。  
**只有接口值才能做类型断言**，具体类型的变量不能写 `x.(T)`。

## 2）两种写法

### ① 单值断言（类型不对会 panic）

```go
var a interface{} = 100
v := a.(int) // 若 a 不是 int：panic
```

### ② 带 `ok` 的安全断言（面试推荐写这种）

```go
v, ok := a.(int)
if ok {
    // 断言成功，v 为 int
} else {
    // 不是 int
}
```

## 3）可以断言成什么？

- **具体类型**：`a.(int)`、`a.(MyStruct)`、`a.(*User)`
- **接口类型**：`r, ok := a.(io.Reader)`（判断动态值是否实现了该接口）

## 4）面试一句话

**类型断言 = 从接口里「拆出」真实类型（或校验是否满足某接口），是接口与具体类型之间的桥梁。**

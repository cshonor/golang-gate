# 空接口与 `eface`（`interface{}` / `any`）

上一篇说完「接口」，`any` / `interface{}` 可以当成 **最宽松的接口**：**没有任何方法**，所以 **所有类型都天然满足它**——就像「不限技能的万能容器」，啥都能装。

> Go 1.18 起 `any` 只是 `interface{}` 的别名，面试说哪个都行。

## 1. 那运行时到底长啥样？

空接口在底层对应 **`eface`**（名字不用死记，知道是 **两根指针** 就行）：

```go
type eface struct {
	_type *_type          // 类型信息：这到底是 int 还是 string 还是 *User
	data  unsafe.Pointer // 数据在哪
}
```

**大白话**：空接口 = **「类型身份证」** + **「数据地址」**。

## 2. 带方法的接口（比如 `io.Reader`）不一样

那种接口底层往往是 **`iface`**（多一个方法表相关的东西），不是单纯的 `eface`。你先把 **空接口 = eface**、**非空接口 = iface** 分开记就够用了。

| | 空接口 | 带方法的接口 |
|--|--------|----------------|
| 底层 | `eface`：`_type` + `data` | `iface`：`itab` + `data`（方法表在 `itab` 里） |
| 典型用途 | `any` 传任意值 | 多态、只关心行为 |

```go
type iface struct {
	tab  *itab
	data unsafe.Pointer
}
```

## 3. 经典坑：「接口不是 nil」≠「里面指针是 nil」

```go
var p *int = nil
var i any = p

fmt.Println(i == nil) // false
```

你可以这样理解：

- 把 `p` 赋给 `i` 时，`i` 已经是一个 **带类型的接口值** 了：类型信息说「我是 `*int`」，数据指针可以是 nil。
- **真正的 `nil` 接口**一般是「类型也没有、数据也没有」那种零值；这里 **类型信息还在**，所以 `i == nil` 为 **false**。

面试说法：**别拿接口去跟裸 `nil` 比着玩**，要分清「整个接口是不是零值」。

---

## 复习速记

| 记什么 | 记一句 |
|--------|--------|
| `eface` | `_type` + `data` |
| nil 坑 | 有类型的 nil 指针装进 `any` 后，接口值 often 仍不是 nil |

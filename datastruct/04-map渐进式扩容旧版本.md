# map 渐进式扩容（旧版本 / 教学提纲）

## 1. 为什么需要扩容

- 元素增多 → 装载因子过高 → 查找/插入变慢。  
- 扩容 = **更多桶**，重新分布 key（rehash）。

## 2. 「渐进式」（incremental）在说什么

- 一次性搬迁所有桶会 **长时间停顿**。  
- Go 采用 **分批搬迁**：在后续 **读写** map 的触发路径上顺带搬一点（具体策略随版本优化，教学上记「分摊到多次操作」）。

## 3. 经典两表模型（概念）

- 扩容过程中可能存在 **旧桶区 + 新桶区**：根据 **搬迁进度** 决定从哪张表读、写时如何迁移。

## 4. 与并发笔记的关系

- 搬迁期间若 **并发写** map → 数据竞争 / 可能 panic。见 `07-map并发不安全.md`。

## 5. 自检

- 渐进式扩容降低的是 **哪一次操作的延迟**？总工作量变了吗？  
- 为什么扩容时遍历更「微妙」（配合 `06-map乱序原理.md`）？

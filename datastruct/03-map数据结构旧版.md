# map 数据结构（旧版 / 经典教学模型）

> 用于对照课程「旧版」讲解；**当前 Go 版本请以 `runtime/map.go` 为准**。

## 1. 高层结构（经典说法）

- **`hmap`**：哈希表头，记录元素个数、桶数量、哈希种子等。  
- **`bmap`（bucket）**：每个桶存 **8 个键值对槽位**（经典 8 槽模型）+ **overflow 指针**（桶链表，处理冲突）。

## 2. 定位键（直觉）

1. 对 key 算哈希，结合随机种子（防 Hash DoS）。  
2. 低位选桶，高位参与桶内 **tophash** 匹配（快速过滤）。  
3. 桶满则沿 **overflow** 链继续找/插。

## 3. `tophash` 的作用（概念）

- 每个槽位存 key 哈希的高字节摘要，减少全 key 比较次数。  
- 特殊标记表示 **空槽、已删除（evacuated）** 等状态（具体取值随版本变化）。

## 4. 与课表的衔接

- 理解「桶 + overflow」后，再接 **渐进式扩容**（`04-map渐进式扩容旧版本.md`）。

# 五、STW（Stop The World）优化：Go GC的核心竞争力

STW是指GC执行时，**暂停所有用户goroutine**，仅执行GC逻辑的阶段，是衡量GC性能的核心指标（STW越短，用户越无感知）。

## 1. STW的必要性

- 标记阶段：需要扫描根对象（goroutine栈、全局变量等），需要与用户程序协调，避免根集合不一致。
- 标记终止阶段：需要处理剩余的灰色对象，完成标记，开启清除阶段。

## 2. Go STW的优化历程（面试必背）

### （1）Go 1.5 并发GC：STW大幅缩短

- STW主要发生在**关键阶段**（如标记开始/结束等，随版本演进细节不同）：
  1. 标记开始：需要建立一致快照、配合写屏障等。
  2. 标记终止：处理剩余灰色对象，完成标记。
- 清除阶段更多以并发方式进行（实现细节随版本变化）。

### （2）Go 1.8 混合写屏障：栈扫描相关停顿进一步可控

- 混合写屏障使并发标记下的一致性更容易维护，整体停顿更短、更稳定。

### （3）Go 1.14 异步抢占：改善「等安全点」带来的长尾停顿

- 之前栈扫描/抢占更依赖协作式安全点，某些循环可能导致停顿不稳定。
- 异步抢占：通过信号等机制在更多时机打断运行，使 GC 相关停顿更可控。

### （4）Go 1.15+ 持续优化

- 优化写屏障开销，减少缓存失效。
- 优化并发清除逻辑，减少GC对用户程序的影响。
- 优化根对象扫描，进一步缩短STW。

## 3. 现在Go的STW水平（经验表述）

- Go 1.14+ 生产环境中，常见场景下 **STW 可稳定在亚毫秒级量级**；堆很大时具体数字仍取决于对象图、根集合、负载与版本，**应以 `runtime/trace` / `pprof` 实测为准**。

# 四、屏障技术（Write Barrier）：并发GC的核心保障

写屏障是**并发GC的必要条件**，作用是在用户程序修改对象引用时，插入额外逻辑，防止漏标。

## 1. 两种基础写屏障

### （1）插入屏障（Dijkstra Barrier）

- **规则**：当一个黑色对象引用一个白色对象时，将白色对象标记为灰色。
- **原理**：保证「黑色对象不会引用白色对象」，避免漏标。
- **缺点**：需要在标记结束后**重新扫描所有goroutine栈**（栈上的写操作不触发写屏障），导致STW时间长。
- **Go 1.5采用**，但栈扫描STW问题严重。

### （2）删除屏障（Yuasa Barrier）

- **规则**：当一个对象的引用被删除时，将被删除的对象标记为灰色，保留到标记结束。
- **原理**：保证「白色对象不会失去所有引用」，避免漏标。
- **缺点**：会保留大量死亡对象，导致GC回收效率低，内存开销大。

## 2. Go 1.8+ 混合写屏障（Hybrid Write Barrier）

Go 1.8引入了**混合写屏障**，结合了插入屏障和删除屏障的思路，缓解并发标记中的漏标问题，是目前Go GC的核心屏障之一。

### 混合写屏障的核心规则（直观版）

对于堆上指针的写操作 `*slot = new`（即把`new`赋值给`slot`指向的内存）：

1. **插入屏障部分**：将`new`（新引用的对象）标记为灰色，保证黑色对象不会直接「挂住」未扫描的白色对象。
2. **删除屏障部分**：将`*slot`（旧引用的对象）标记为灰色并保留到标记结束，避免旧引用链上的对象被过早漏标。

### 混合写屏障的核心优势（面试表述）

- **降低栈相关 STW**：配合并发标记与抢占，使整体停顿更可控。
- **与并发标记协同**：写屏障是并发 GC 正确性的关键拼图之一。
- **开销**：有，但相对可控；具体与版本、负载相关。

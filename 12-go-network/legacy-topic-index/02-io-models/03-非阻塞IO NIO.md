# 非阻塞 IO（NIO）

> **02-io-models · IO 模型全解**  
> 上篇：[02-阻塞IO BIO](./02-阻塞IO%20BIO.md) · 前置：[02-阻塞与非阻塞](../01-io-fundamentals/02-阻塞与非阻塞.md)

---

## 一、核心定义

对 socket 设置 **`O_NONBLOCK`**（或创建时即非阻塞）后：**数据未就绪**时，`read`/`recv` **立即返回**，常见错误码 **`EAGAIN` / `EWOULDBLOCK`**（具体以平台文档为准），**调用线程不会因「等一个字节」而睡眠**。

就绪后，同一次 `read` 往往很快把已有数据拷到用户空间——在教材分类里，多路复用 + 非阻塞读整体仍常归为 **「同步 I/O」**（完成读的是用户态发起的 `read`）。

### 典型流程（读）

1. `read` 发现无数据 → 立刻 **`EAGAIN`**。  
2. 若**没有**多路复用，只能 **轮询** 或 `sleep` 降频（折中延迟）。  
3. 有数据后 `read` 返回正字节数。  
4. 生产上：**非阻塞 fd + `epoll_wait` / `poll` 等**，而不是死循环 `read`。

---

## 二、特点

### 优点

- **等就绪阶段线程不阻塞在 `read` 上**（可配合单线程事件循环）。  
- 与多路复用结合后，**避免「一线程一连接」**。

### 缺点

- **裸轮询**：CPU 空转、syscall 风暴。  
- **代码分支多**：`EAGAIN`、重试、与 buffer 协议结合。  
- **很少单独作为架构终点**：几乎总配 **I/O 多路复用**。

---

## 三、注意点

1. **非阻塞 ≠ 异步**（见 [03-同步与异步](../01-io-fundamentals/03-同步与异步.md)）。  
2. **别 busy-loop**：无复用器时至少加退避或 `poll` 间隔。  
3. **Go**：网络 fd 底层默认非阻塞，是 **`netpoll` 的前置条件之一**。

---

## 四、典型场景

- **`epoll`/`kqueue`/IOCP** 等事件源背后的 fd 形态。  
- 自研 Reactor / 与语言 runtime 集成的底层。

---

## 五、极简总结

- NIO：**未就绪立刻返回**，靠 **`EAGAIN`** 与上层策略协作。  
- 与多路复用是 **标配组合**；裸轮询仅作反例理解。  
- Go：**非阻塞 + netpoll**。

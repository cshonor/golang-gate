# 五大 IO 模型总览

> **02-io-models · IO 模型全解**  
> 前置：[01-什么是 IO](../01-io-fundamentals/01-什么是IO.md)、[02-阻塞与非阻塞](../01-io-fundamentals/02-阻塞与非阻塞.md)。本篇给 **Stevens 五模型** 地图，后文逐篇展开。

---

## 一、核心背景

网络 IO（可推广到广义阻塞读写的 fd）常被拆成两个阶段：

1. **等数据就绪**（数据是否在**内核**侧可读/可写）  
2. **把数据拷到用户空间**（或反向写出）

围绕这两阶段如何**阻塞线程**、如何**通知**、由**谁拷贝**，演化出五种经典模型：

1. **阻塞 IO（Blocking IO，BIO）**  
2. **非阻塞 IO（Non-blocking IO，NIO）**  
3. **IO 多路复用（I/O Multiplexing）**（`select` / `poll` / `epoll` 等）  
4. **信号驱动 IO（Signal-driven I/O）**  
5. **异步 IO（Asynchronous I/O，AIO / `io_uring` 等）**

---

## 二、核心区分维度（面试必背）

| 维度 | 含义 |
|------|------|
| **阻塞 / 非阻塞** | **等就绪**阶段，调用线程是否会在该调用上**睡眠**（见 [02](../01-io-fundamentals/02-阻塞与非阻塞.md)） |
| **同步 / 异步** | **谁完成拷贝 / 谁拿结果**；与阻塞非阻塞**正交**（见 [03-同步与异步](../01-io-fundamentals/03-同步与异步.md)） |
| **等待方式** | 阻塞等、轮询、`epoll_wait` 等、信号、完成队列 |
| **并发与成本** | 单线程能盯多少 fd、线程数、syscall 次数、CPU 空转 |

## 三、五大模型速览

### 1. 阻塞 IO（BIO）

- **行为**：等就绪 + 拷贝，**调用线程全程阻塞在这一次系统调用上**。  
- **特点**：写代码最省心；**一线程一连接**时扩展性差。  
- **篇**：[02-阻塞IO BIO](./02-阻塞IO%20BIO.md)

### 2. 非阻塞 IO（NIO）

- **行为**：未就绪立刻 **`EAGAIN`** 类返回；就绪后同一次 `read` 往往很快完成拷贝。  
- **特点**：线程不必睡在「等一个字节」上，但**裸轮询**会烧 CPU；生产上几乎总配 **多路复用**。  
- **篇**：[03-非阻塞IO NIO](./03-非阻塞IO%20NIO.md)

### 3. IO 多路复用

- **行为**：**一次阻塞**在复用器上，等**多个 fd 中至少一个就绪**，再对每个就绪 fd 做 `read/write`。  
- **特点**：高并发主流；Linux 上 **`epoll`** 是代表。  
- **篇**：[04-select](./04-IO多路复用-select.md) → [05-poll](./05-IO多路复用-poll.md) → [06-epoll](./06-IO多路复用-epoll原理.md) → [07-LT/ET](./07-epoll-LT与ET模式.md)

### 4. 信号驱动 IO

- **行为**：注册信号，就绪时内核发信号，用户在信号处理里读。  
- **特点**：生产少见；限制多。  
- **篇**：[08-信号驱动IO](./08-信号驱动IO.md)

### 5. 异步 IO

- **行为**：提交请求后内核在后台等+拷，完成再以事件/回调通知（真异步语义依实现）。  
- **特点**：上限高；复杂度高、依赖 OS。  
- **篇**：[09-异步IO AIO](./09-异步IO%20AIO.md)

## 四、Go 中的定位（鸟瞰）

Go 网络栈常见形态：**非阻塞 socket + 多路复用（Linux 上多为 `epoll`）+ `runtime/netpoll`**，对上仍提供 **`Read`/`Write` 的「同步阻塞式」写法**——阻塞的是 **G**，不是「每个连接占满一个 OS 线程」。详见 **`07-go-netpoll`** 与 **06** 的 `poll` 桥梁篇。

## 五、学习顺序建议

1. [BIO](./02-阻塞IO%20BIO.md) / [NIO](./03-非阻塞IO%20NIO.md)：把「等就绪」与 **`EAGAIN`** 吃透。  
2. **多路复用**：`select` → `poll` → **`epoll`** → **LT/ET**。  
3. [信号驱动](./08-信号驱动IO.md)：知道边界即可。  
4. [异步 / io_uring](./09-异步IO%20AIO.md)：与 epoll 的**语义差异**。  
5. [对比总结](./10-五大IO模型对比总结.md)：收束成一张嘴能讲清的表。

---

## 六、极简总结

- 五模型差异 = **等就绪 + 拷贝** 两阶段怎么分工。  
- 生产主流：**多路复用 + 非阻塞 fd**；Go 再叠 **G 调度**。  
- **真异步**看 OS（`io_uring` / IOCP 等）与 runtime 演进。  
- **信号驱动**、**裸 NIO** 很少单独当主架构。

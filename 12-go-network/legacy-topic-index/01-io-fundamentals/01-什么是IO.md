# 什么是 IO

> **01-io-fundamentals · IO 核心理论**：数据如何在用户态与外部世界之间搬运；为 [02-阻塞与非阻塞](./02-阻塞与非阻塞.md)、[03-同步与异步](./03-同步与异步.md)、多路复用与零拷贝打地基。

IO，全称 **Input/Output**，本质可以压成一句：

**数据在「内存」与「外部设备」之间的搬运过程。**

这里的「内存」通常指 **进程的用户态地址空间**；「外部世界」包括：磁盘文件、网卡（网络）、键盘、显示器、管道、socket 等——凡是不在当前进程地址空间里、又要参与数据交换的，都算 IO 端点。

---

## 一、IO 的核心定义

一次标准 IO 行为，就是：

- 从外部设备把数据读到内存（**Input / Read**）
- 把内存里的数据写到外部设备（**Output / Write**）

操作系统不允许用户进程直接按物理地址操作硬件，因此必须经过 **内核** 中转：

1. 进程发起 `read` / `write`（或同类封装）**系统调用**
2. 内核与设备驱动、协议栈、页缓存等协作，完成真正 I/O
3. 内核把数据拷进用户缓冲区，或从用户缓冲区取走

因此，从模型上记：**一次 IO ≈ 跨边界的数据移动 + 内核路径上的检查与调度 + 设备/链路上的等待**（具体几次拷贝、是否阻塞，见 [04-用户态与内核态](./04-用户态与内核态.md)、[05-缓冲区、页缓存、零拷贝](./05-缓冲区、页缓存、零拷贝.md)）。

### 关键边界：用户空间 vs 内核空间

- **用户空间**：进程自己的虚拟地址空间，应用代码与 `buf []byte` 等用户缓冲都在这里。  
- **内核空间**：内核代码与内核数据结构（协议栈缓冲、页缓存、驱动环形缓冲等）所在区域，**受特权级保护**。  
- **硬约束**：用户进程**不能绕过内核直接操作网卡/磁盘控制器**；所谓「做 IO」在 POSIX 上几乎总是 **`read` / `write` 等系统调用** 与内核协作。

---

## 二、IO 的分类

### 1. 按数据方向

- **输入（Input）**：外部 → 内存，例如 `read`、`recv`、`accept`（`accept` 得到的是新连接描述符，语义上仍是「从内核取结果」）  
- **输出（Output）**：内存 → 外部，例如 `write`、`send`

### 2. 按操作对象

- **文件 IO**：普通文件、块设备、标准输入输出等  
- **网络 IO**：socket、TCP/UDP 字节流  
- **设备 IO**：键盘、串口等字符设备  

在 Unix/Linux 里习惯说 **一切皆文件**：许多端点在 API 层都落到 **fd + read/write**，便于统一心智（细节差异见 [06-文件IO vs 网络IO](./06-文件IO%20vs%20网络IO.md)）。

### 3. 按阻塞 / 非阻塞、同步 / 异步（后文展开）

- **阻塞 / 非阻塞**：未就绪时，**调用是否会让当前线程睡眠**（见 [02-阻塞与非阻塞](./02-阻塞与非阻塞.md)）。  
- **同步 / 异步**：**完成语义**如何落在调用边界上、由谁通知完成（见 [03-同步与异步](./03-同步与异步.md)）。  
- **五大 IO 模型**（BIO、NIO、多路复用、信号驱动、异步）的系统化地图见：[五大 IO 模型总览](../02-io-models/01-五大IO模型总览.md)。

## 三、从系统调用看一次 `read`

1. 应用调用 `read(fd, buf, n)`  
2. CPU 进入**内核态**，由内核在该 `fd` 上执行读路径  
3. 内核判断数据是否已在可用路径上（例如已在内核 socket 接收缓冲区、或页缓存命中）  
   - **已就绪**：拷贝到用户态 `buf`，返回字节数  
   - **未就绪**：依 IO 模式——**阻塞**直到有数据，或**非阻塞**返回 `EAGAIN` 等（见 [02-阻塞与非阻塞](./02-阻塞与非阻塞.md)）  
4. 返回用户态，应用继续执行  

要点：**IO 往往不是「一次函数调用就必然结束」**；中间可能有等待、多次拷贝、多段内核逻辑。

---

## 四、IO 为什么慢？慢在哪里？

IO 常常是整体延迟的大头，典型来源：

1. **设备与链路本身慢**：磁盘、广域网 RTT 与带宽相对内存数量级更差。  
2. **用户态 ↔ 内核态切换与拷贝**：每次系统调用有陷入/返回成本；跨边界拷贝占 CPU 与内存带宽（见 [04-用户态与内核态](./04-用户态与内核态.md)、[05-缓冲区、页缓存、零拷贝](./05-缓冲区、页缓存、零拷贝.md)）。  
3. **等待就绪**：数据未到、对端未写、缓冲区满等，调用方要么**挂起**（阻塞），要么**轮询/交给多路复用器**（非阻塞 + `select`/`poll`/`epoll` 等，见 [02-io-models](../02-io-models/01-五大IO模型总览.md)）。

## 五、与后续章节的关系

整本 IO 模型都在回答同一件事：**一次 IO 调用发生以后，由谁等、等多久、数据拷几次、如何同时盯多个 fd**。

| 主题 | 解决什么问题 |
|------|----------------|
| [阻塞与非阻塞](./02-阻塞与非阻塞.md) | 数据未就绪时，**调用是否立刻返回** |
| [同步与异步](./03-同步与异步.md) | **谁来等完成**、完成如何**通知**（与阻塞/非阻塞正交，勿混为一谈） |
| [多路复用](../02-io-models/01-五大IO模型总览.md) | **一组 fd** 上谁就绪了再读/写 |
| 零拷贝 / 缓冲 | 减少**拷贝与系统调用次数** |

抓住：**IO = 跨空间搬运数据**；后面所有「高级模型」都是在优化 **怎么等、谁等、拷几次、如何并发更高**。

---

## 极简总结（笔记末尾可剪贴）

- IO = 数据在 **内存 ↔ 外部设备** 之间搬运。  
- 必须经过 **内核路径**，不能指望用户进程直接碰硬件。  
- 慢在：**设备/链路延迟**、**拷贝**、**等就绪**。  
- 阻塞、非阻塞、多路复用、异步，都是围绕 **同一次 IO 语义** 的不同策略组合。

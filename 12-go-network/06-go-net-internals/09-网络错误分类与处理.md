# 网络错误分类与处理

> **06-go-net-internals · Go net 包源码级理解**  
> 与 [02-阻塞与非阻塞](../01-io-fundamentals/02-阻塞与非阻塞.md) 中的 **`EAGAIN`** 语义、`poll` 返回路径一起理解效果最好。

---

## 1. 错误从哪来

- **syscall 层**：`ECONNREFUSED`、`ECONNRESET`、`EPIPE`、`ETIMEDOUT` 等。  
- **`net` 封装层**：常包装为 **`net.OpError`**（带 `Op`、`Net`、`Source`、`Addr`、`Err`）。  
- **语义层**：`io.EOF`（对端正常关写、本端读尽等场景）、`io.ErrUnexpectedEOF` 等。

---

## 2. `net.Error` 接口（面试常问）

```go
type Error interface {
	error
	Timeout() bool   // 是否为超时类
	Temporary() bool // 是否「可能重试」（Go 1.18+ 已弃用提示见 release note，新代码勿依赖过深）
}
```

实践建议：

- **`Timeout()`**：走 **退避重试 / 切换链路 / 返回 504 类** 策略。  
- **连接类硬错误**（如 refused、reset）：**重试通常无意义**，应上报并熔断或换实例。  
- **`Temporary()`**：仅作**弱提示**；是否重试要结合业务幂等与上限。

---

## 3. 典型错误一览（速查）

| 现象 | 常见底层 | 处理方向 |
|------|-----------|----------|
| 对端关连接 / 读尽 | `EOF` / `ECONNRESET` | 上层结束循环、回收连接 |
| 本端写已关的连接 | `EPIPE` / broken pipe | 检查生命周期、避免并发写 |
| 超时 | `i/o timeout`，`Timeout()==true` | 调整 deadline、拆分读写 SLA |
| 非阻塞暂无数据 | `EAGAIN` | 由 runtime/netpoll 消化，业务 `Read` 多表现为阻塞语义或重试 |

---

## 4. 最佳实践（工程）

1. **`errors.Is` / `errors.As`**：unwrap `OpError` 里的 `syscall.Errno`。  
2. **日志**：打印 **`Op` + `Net` + `Addr` + 根因 `Err`**，线上排障效率最高。  
3. **重试**：只对 **幂等、可安全重复** 的操作；加 **次数上限 + 总时长上限**。  
4. **与 Context**：超时往往双轨：**context 取消** + **deadline**；对齐产品语义，避免只设一边。

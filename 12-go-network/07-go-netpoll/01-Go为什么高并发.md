# Go 为什么高并发

> **07-go-netpoll · Go netpoll 高并发核心**

## 内容大纲

- G-M-P：goroutine 轻量、阻塞 syscall 时线程剥离（大方向）
- netpoll：把网络 fd 就绪与调度 goroutine 衔接
- 对比线程 per connection：内存与上下文切换
- 与 GC、栈扩容的边界（点到为止）
- 误区：goroutine 不是免费；仍需背压与池化

## 扩写指引（闭环）

把 **「高并发」** 说成一句话：**非阻塞 fd + netpoll + G 挂起/唤醒** 替代 **一线程一阻塞**。完整链路见 **[08-netpoll与GMP调度深度联动](./08-netpoll与GMP调度深度联动.md)**；**`net` 侧入口**见 **06** [07-pollDesc核心结构与原理](../06-go-net-internals/07-pollDesc核心结构与原理.md)。

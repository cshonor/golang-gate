# slice 扩容

## 1. 何时扩容

- `append` 后 **长度超过当前容量** → 分配 **新底层数组**，把旧元素拷过去，**slice 头指向新数组**。

## 2. 新容量怎么定（规则随版本调整）

- 小切片往往 **翻倍** 左右增长；变大后系数会 **减小**（减少内存浪费），具体以 `runtime/slice.go` 中 `growslice` 为准。  
- **元素大小**、是否包含指针等会影响策略（GC 扫描相关）。

## 3. 与「共享切片」的关系

- 扩容前：**多个 slice 可能共享旧数组**。  
- 扩容后：**当前 slice** 往往指向新数组，**旧兄弟 slice 仍指向旧数组** → 易错点。

## 4. 使用建议

- 已知最大长度时用 `make([]T, 0, n)` **预分配**，减少反复扩容。  
- 大切片 `append` 后若长期只保留一小段，注意 **底层数组仍占内存**（可 `copy` 到新小切片释放大数组，视场景）。

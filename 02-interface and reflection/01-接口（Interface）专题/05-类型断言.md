# 类型断言（Type Assertion）

前面说过：接口变量 **静态类型**是接口，**运行时**才知道「背后到底是 Dog 还是 Cat」。  
**类型断言**就是：**把接口里的「真身」拆出来**，或者试试能不能当成另一个接口用。

> 只有 **接口类型的值** 才能写 `x.(T)`；普通变量不能断言。

## 1. 先用大白话想：为什么叫「拆包」？

接口变量在「接口的世界里」只知道契约上的方法（比如只会 `Speak()`），**不知道**你背后有没有 `Bite()` 这种 **具体类型专属** 的方法。  
要用专属能力，就得 **断言回具体类型**：

```go
type Speaker interface {
	Speak() string
}

type Dog struct{ Name string }

func (d Dog) Speak() string { return d.Name + " 汪汪" }
func (d Dog) Bite() string   { return d.Name + " 咬人了" }

type Cat struct{ Name string }
func (c Cat) Speak() string { return c.Name + " 喵喵" }

var s Speaker = Dog{Name: "旺财"}

fmt.Println(s.Speak())        // OK，接口里声明过
// fmt.Println(s.Bite())     // 编译不过：Speaker 没有 Bite

dog, ok := s.(Dog)
if ok {
	fmt.Println(dog.Bite()) // OK，dog 已经是具体类型
}
```

## 2. 两种写法：猛男版 vs 稳妥版

**不带 ok**：错了就 **panic**（适合你很确定类型的时候）。

```go
var a any = 100
v := a.(int)
```

**带 ok**：面试、生产代码都更推荐。

```go
v, ok := a.(int)
if ok {
	// v 是 int
}
```

## 3. 能断言成啥？

- **具体类型**：`a.(int)`、`a.(MyStruct)`、`a.(*User)`
- **另一个接口**：`r, ok := x.(io.Reader)`，看动态值是否满足那个接口

## 4. 面试一句话

**类型断言** = 从接口值里 **取出/校验** 动态类型（具体类型或另一接口），是接口和具体类型之间的桥。

## 复习速记

| 记什么 | 记一句 |
|--------|--------|
| 语法 | `x.(T)`，`x` 必须是接口值 |
| 安全 | `v, ok := x.(T)` |
| 干啥 | 拆包、转接口 |

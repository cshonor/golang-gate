# 接口 + 反射：2 道面试真题与标准答法

## 真题 1：为什么 `var i interface{} = (*T)(nil)` 时，常常 `i != nil`？

**考点**：`eface` 两字段、接口零值。

**标准答法**：

- 接口值在实现里可以看成「类型信息 + 数据指针」；空接口即 `eface`。
- 把**有类型的 nil 指针**赋给接口时，**类型信息不为空**（例如 `*T`），数据指针可以为 nil。
- **接口值为 nil** 一般要类型与数据都「空」；这里类型已确定，所以整个接口值**不是** nil。
- 和业务里「判断接口是否为 nil」的 bug 相关：应显式判断具体类型/用反射/避免把「nil 指针」塞进接口再和裸 `nil` 比。

## 真题 2：下面哪段能合法通过反射把 `x` 改成 `30`？不能的话为什么？

```go
var x int = 10
v := reflect.ValueOf(x)
v.SetInt(30)
```

**考点**：`CanSet`、可寻址、`Elem`。

**标准答法**：

- **不能**。`ValueOf(x)` 得到的是整型值的**副本**，不可设置，调用 `SetInt` 会 **panic**。
- 正确写法：`reflect.ValueOf(&x).Elem().SetInt(30)`，先取指针再 `Elem()`，得到代表 `x` 本身的、可设置的 `Value`。
- 补充：若 `x` 来自 map、接口等不可寻址场景，即使指针也要看是否能拿到可设置路径；**未导出字段**在包外不能改。

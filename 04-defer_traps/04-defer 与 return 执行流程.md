# 04 - defer 与 return 执行流程

`return` 与 `defer` 的先后关系是面试高频点：**不是**「先跑完整个 `return` 表达式再 defer」，而是 **`return` 先把结果写入返回路径，再按 LIFO 执行 `defer`，最后才真正返回**。LIFO 总览见 [02](./02-defer%20执行顺序规则.md)；实参何时求值见 [03](./03-defer%20参数预计算陷阱.md)。

---

## 1. 执行顺序（官方语义，记三拍）

对一次 `return` 可记成三拍：

1. **写入返回值**（具名返回：写入具名变量；匿名返回：把值拷贝到返回槽/临时路径——教学模型下可理解为「先确定要返回的值」）。
2. **执行所有 `defer`（LIFO）**。
3. **函数退出**，把第 1 步确定的结果交给调用方。

一句话：

> **先写入返回路径 → 再 defer → 最后真正返回。**

---

## 2. 两种返回值：命名 vs 匿名（面试必考）

### A. 命名返回值（Named return values）

- 返回值变量在函数入口就存在（可理解为栈上的「结果槽」）。
- `return expr` 会把 `expr` 赋给该变量。
- `defer` 里可以继续改这个变量，**最终对外返回的是变量最终值**。

```go
func f() (ret int) {
	defer func() { ret++ }()
	return 1
	// 1) ret = 1
	// 2) defer: ret = 2
	// 3) 返回 2
}
```

### B. 匿名返回值（Unnamed return values）

- `return expr` 常见教学理解：先把 `expr` 的结果放进「返回槽」，再跑 `defer`。
- `defer` 里改的是**局部变量**，通常**改不到**已经写入返回槽的那份值。

```go
func f() int {
	ret := 1
	defer func() { ret++ }()
	return ret
	// 1) 返回槽 ≈ 1
	// 2) defer: ret=2（局部）
	// 3) 仍返回 1
}
```

> 细节以实现为准；工程上记住：**想靠 defer 改返回值，用命名返回；否则别指望 defer 改匿名返回的“结果”。**

---

## 3. 底层直觉简图

### 命名返回值

```
[ ret 结果槽 ]  ← defer 直接改这里
return 1 → ret=1 → defer → ret=2 → 对外返回 2
```

### 匿名返回值（教学模型）

```
局部 ret
return ret → 值进入返回槽（1）
defer 改 ret（2）→ 返回槽仍是 1 → 对外返回 1
```

---

## 4. 经典面试题（含答案）

### 题 1：命名返回值

```go
func a() (x int) {
	defer func() { x += 10 }()
	return 5
}
```

**答案：`15`**（`x=5` → defer `x+=10`）。

### 题 2：匿名返回值

```go
func b() int {
	x := 5
	defer func() { x += 10 }()
	return x
}
```

**答案：`5`**（返回槽先固定为 5，defer 改局部 `x` 不影响返回槽）。

### 题 3：多个 defer（LIFO）

```go
func c() (x int) {
	defer func() { x++ }()
	defer func() { x += 2 }()
	return 10
}
```

**答案：`13`**：`defer` **后注册先执行**，因此先执行 `x+=2`（12），再执行 `x++`（13）。

---

## 5. 项目实践建议

**命名返回 + defer 改返回值的优点**

- 适合短工具函数：在边界统一补错误码、补默认值、做收尾修正。

**缺点 / 风险**

- 长函数里「隐式改返回值」可读性差，review 很难一眼看出最终返回是什么。

**实践建议**

- **短函数 / 框架边界**可谨慎使用命名返回 + defer。
- **复杂业务逻辑**优先匿名返回 + 显式 `return v, err`，把意图写清楚（更多反模式见 [08](./08-defer%20最佳实践与反模式.md)）。

---

## 6. 一句话总结

- **命名返回**：`return` 写入结果槽 → `defer` 还能改槽 → 返回最终值。
- **匿名返回**：`return` 先固定返回槽 → `defer` 通常改不到槽 → 返回原值。
- **`defer` 夹在「写入返回路径」与「函数退出」之间。**

---

## 速背一页纸

| 点 | 记法 |
|----|------|
| 顺序 | 写入返回 → defer（LIFO）→ 退出 |
| 命名 | defer 可改返回值变量 |
| 匿名 | defer 一般改不了已确定的返回值 |
| 多 defer | **后注册先执行** |
| 风险 | 长函数别靠 defer 偷偷改返回 |

**题**：`a()`→15，`b()`→5，`c()`→13。

# 七、面试高频真题（含标准答案）

## 1. 问：Go的内存分配原理是什么？堆和栈的区别？

**答**：

Go的内存分为栈和堆：

- **栈**：goroutine私有，存储局部变量，函数返回后随栈帧释放，**不经过 GC 回收堆对象**；采用动态扩容的连续栈设计。
- **堆**：全局共享，存储需要更长生命周期/共享逃逸的对象，由GC管理；采用 `mcache` / `mcentral` / `mheap` 等分层分配器，小对象优先走 per-P 缓存以降低锁竞争。
- **区别**：栈分配对 GC 压力更小；堆对象由 GC 跟踪；栈是 per-goroutine，堆是共享；生命周期与逃逸分析强相关。

## 2. 问：什么是逃逸分析？怎么判断变量是否逃逸？

**答**：

逃逸分析是Go编译器的静态分析，用于判断变量是否逃逸出函数栈，决定分配到栈还是堆。

- **常见逃逸场景**：返回局部指针、动态大小 `make`、闭包捕获、接口装箱、发送到 channel、全局引用等（以 `-gcflags=-m` 为准）。
- **查看方法**：`go build -gcflags="-m -l" main.go`。
- **优化方向**：减少堆分配、降低接口/指针返回、复用对象（`sync.Pool` 等）。

## 3. 问：Go的GC算法是什么？三色标记清除的原理？

**答**：

Go采用并发三色标记清除（追踪式 GC）：

- **白色**：未处理；**灰色**：已发现，子引用未扫完；**黑色**：自身与子引用已处理完。
- **流程**：从根集合出发标记可达对象，最终不可达对象回收。
- **并发标记**需要写屏障等机制避免并发修改指针导致的漏标/不一致。

## 4. 问：Go为什么用混合写屏障？原理是什么？

**答**：

并发标记时，用户程序会不断修改指针；需要写屏障记录这些变化，保证标记阶段的一致性。

- **混合写屏障**同时处理「新引用」与「旧引用」相关的标记需求，配合并发标记降低整体 STW，并保证正确性（细节以实现为准）。
- **优势**：相比早期方案，更容易把全局停顿控制在更低水平。

## 5. 问：怎么优化Go的GC性能？

**答**：

1. **核心**：减少堆分配（逃逸分析、`sync.Pool`、减少装箱与指针逃逸）。
2. **调参**：`GOGC`、`GOMEMLIMIT`（容器场景）。
3. **观测**：`pprof`、`runtime/trace`，定位分配热点与 GC 周期。
4. **工程**：避免泄漏（goroutine/channel/map 缓存无限增长）。
5. **版本**：升级 Go 以获取 GC/分配器/抢占等改进。

# 三、Go GC核心算法：三色标记清除（Tri-color Mark-and-Sweep）

Go从1.5版本开始采用**并发三色标记清除算法**，是目前Go GC的核心基础，面试100%会问。

## 1. 三色标记的基础原理

三色标记是一种**追踪式GC**，通过给对象染色来标记存活对象，清除死亡对象：

- **白色对象**：未被访问过的对象，初始状态所有对象都是白色。
- **灰色对象**：已被访问过，但子节点（引用的对象）还未被标记的对象。
- **黑色对象**：已被访问过，且所有子节点都已标记的对象（存活对象）。

### 标记流程（串行版）

1. **初始阶段**：所有对象标记为白色，将根对象（全局变量、goroutine栈、寄存器等）标记为灰色，放入灰色队列。
2. **标记阶段**：从灰色队列取出对象，标记为黑色，将其引用的所有白色对象标记为灰色，放入队列，重复直到灰色队列为空。
3. **清除阶段**：遍历所有对象，清除所有白色对象（死亡对象），黑色对象保留。

## 2. 并发GC的核心问题：漏标（Missing Mark）

串行三色标记全程STW，不会有问题，但**并发GC**（标记阶段和用户程序同时执行）会出现漏标：

- 场景：一个黑色对象A，原本引用白色对象B，用户程序在标记过程中，让A引用了白色对象C，同时删除了A对B的引用。
- 结果：C是白色，没有被标记（A已经是黑色，不会再被扫描），可能导致存活对象未被标记。

**解决漏标需要写屏障（Write Barrier）等机制配合**，见 [06-写屏障.md](./06-写屏障.md)。

## 3. Go GC的演进历程（面试必背）

Go的GC一直在围绕「减少STW、提升并发、降低开销」优化，核心版本演进：

| Go版本 | GC核心优化 | STW时间（量级，实际因负载而异） |
|--------|------------|--------------------------------|
| 1.3及之前 | 串行标记清除，全程STW | 数百毫秒~秒级 |
| 1.5 | 并发三色标记+插入写屏障 | 数十毫秒 |
| 1.8 | 混合写屏障（Hybrid Write Barrier） | 亚毫秒级 |
| 1.14 | 异步抢占（Asynchronous Preemption） | 微秒级（常见 <1ms） |
| 1.15+ | 并发清除优化、写屏障开销降低 | 稳定亚毫秒级 |
| 1.17+ | 寄存器ABI、栈扫描优化 | 进一步降低STW |

# 阻塞与唤醒的代价

> 课表：**锁实现原理 · 基础版** — 阻塞唤醒的代价

## 1. 上下文切换

- 阻塞往往伴随 **从用户态陷入内核**、调度器选择下一个 runnable、切换寄存器/栈 → **微秒级** 开销常见。  
- 高频锁竞争 → **切换风暴**，比短自旋还贵。

## 2. 缓存局部性

- 被换下的线程稍后继续跑时，**cache 可能已冷**，临界区前后代码变慢。

## 3. 延迟抖动

- 唤醒时机依赖调度策略与系统负载 → **尾延迟** 变差（P99 升高）。

## 4. 设计启示

- **减少临界区**、**降低锁竞争**（细粒度锁、分片、无锁队列等）比「选更牛的锁」更根本。  
- 用户态 **futex** 类机制尽量减少 **无谓陷入内核**（下一节）。

## 5. 自检

- 「阻塞」一定进入内核吗？（思考：纯用户态调度器上的协程）  
- 什么时候 **宁愿阻塞** 也不要再自旋？

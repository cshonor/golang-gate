# 锁实现 2：自旋锁（spin lock）

> 课表：**锁实现原理 · 基础版** — 自旋锁

## 1. 行为

- 获取锁失败时 **不睡眠**，在 CPU 上 **循环重试**（常配合 `atomic` / CAS）。  
- 锁 **很快释放** 时，可避免线程阻塞与上下文切换。

## 2. 典型实现直觉

```text
while !CAS(lock, 0, 1) {
    // spin：空转或 PAUSE 指令降低功耗/流水线冲突
}
```

- **释放**：`store(lock, 0)` 或原子写回（需与架构内存序一致）。

## 3. 适用场景

- 临界区 **极短**、竞争 **不极端**（内核短路径、部分 runtime 内部）。  
- 用户态业务层长时间自旋通常 **不划算**。
